## 客観的観察（断定はしない、解析は侍・隠密に委ねる）

1. **`barDesiredSize`と`barActualSize`が全ケースで乖離**（行1のみ一致）。`barActualSize`は8回すべて
   `(540.00,38.00)`で一定（見た目上のバーサイズと一致）だが、`barDesiredSize`は呼び出しごとに
   `(540,38)` `(839.59,257.57)` `(839.59,359.47)` `(877.39,257.57)` の間で変動している。
2. **クランプ計算は`barDesiredSize`を用いている**（`clampMax = workAreaOrigin + workAreaSize - barDesiredSize`
   の関係が全行で数値的に一致）。`barActualSize`（実際の見た目サイズ）ではなくこの値を使っているため、
   実サイズ38に対し最大359.47もの過大な値でクランプ計算され、本来クランプが不要な位置でも押し上げられる。
3. **同一セル（行10）を3回開き直した際、`barDesiredSize.Height`が`257.57`⇔`359.47`の2値を往復**（1回目
   257.57、2回目359.47、3回目257.57）。`translateOut`・`preClamp`（セル自体の位置）は3回とも完全に
   同一値であり、セル位置の計算自体はブレていない。ブレの発生源は`barDesiredSize`の値のみ。
4. `barDesiredSize.Width`もセルの列位置・ズーム倍率によって変動している（540→839.59→877.39）。
