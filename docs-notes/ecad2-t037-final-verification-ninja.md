## 検証環境メモ（運用上の気づき）

マルチモニタ環境でUI Automationの`BoundingRectangle`と実際のマウスクリック座標が
一致しない場面があった（部品選択リストの項目クリックで1行分ズレる事象）。原因は
未特定だが、対策として「クリック直前に対象要素の座標を再取得する」「クリック後は
必ずステータスバー等で意図した要素が選択されたか確認する」を徹底することで実害なく
検証を完遂できた。モーダルダイアログへのキー入力は、`Set-Ecad2Foreground`（メイン
ウィンドウハンドル基準）では届かず、ダイアログ自身のネイティブウィンドウハンドルを
UI Automation経由で取得し`SetForegroundWindow`する必要があった。
