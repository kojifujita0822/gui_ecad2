## 観点別 OK/NG

### 1. シート追加 — OK
- 濃紺スタート（Sheets=0）から「シート1」を追加 → キャンバスが作業領域色（白+グリッド）へ
  正しく切替、NotifyCurrentSheetChangedの0→1遷移明示発火が機能していることを確認。
- 2枚目・3枚目追加時も、常に新規シートが自動選択される（AddCommandの意図通り）。

### 2. シート削除の境界（先頭/中間/末尾/下限） — OK（4パターンすべて）
一意名（AAA/BBB/CCC等）で構成し直し、削除後は「名前変更」ダイアログの表示値で
SelectedSheetの実値をデータ側から確認（UI Automationの見た目だけに頼らない）。

- **末尾削除**: シート1,2,3構成・シート3(末尾)選択中に削除 → シート2(新末尾)が選択。
  期待値 `Min(2, Count-1=1)=1` と一致。
- **中間削除**: シート1,2,3構成・シート2(中間)選択中に削除 → 元シート3(繰り上がり)が選択。
  期待値 `Min(1, Count-1=1)=1` と一致（名前変更ダイアログで実値確認）。
- **先頭削除**: AAA/BBB/CCC構成・AAA(先頭)選択中に削除 → BBB(新先頭)が選択。
  期待値 `Min(0, Count-1=1)=0` と一致。
- **下限（2枚→1枚）**: 2枚から1枚への削除は正常に成功。1枚の状態からの追加削除は
  実際にはブロックされる（後述の気づき参照）。

### 3. シート改名 — OK
BBB→DDDへ改名 → リスト内の位置・選択ハイライトとも保持されたまま名前のみ更新。回帰なし。

### 4. DRCジャンプ — OK
DDDシートにコイル(Y1)を孤立配置 → 設計チェック実行で「コイルがあるが接点なし（死にリレー）」
警告(P2行1)発生。CCCシートへ切替後、出力パネルの警告行をクリック →
DDDシートへ1回で切替 + 該当セル選択 + 右パネルのプロパティ(種別:コイル/デバイス名:Y1)表示が
同期。二重発火・表示の食い違いなし。

## 範囲外検出（修正不要・報告のみ）

- **DeleteSheetButtonのUI Automation `IsEnabled`表示が、Sheets.Count=1の状態でも
  `True`と観測された**（本来はDeleteCommandのCanExecute=`Sheets.Count>1`によりfalseになるべき）。
  ただし**物理マウスクリックで実際に押下しても削除は実行されず、1枚のまま維持**された
  （データ側の防御は機能）。CommandManager.RequerySuggestedの再評価がUI Automationの
  IsEnabledプロパティへ反映されるタイミングの問題と推測され、UI Automation観測特有の
  限界の可能性が高い（ninja.mdの既知の罠と類似）。実装バグと断定はできないため、
  気づきとして記録するに留める。念のため気に留められたし。
