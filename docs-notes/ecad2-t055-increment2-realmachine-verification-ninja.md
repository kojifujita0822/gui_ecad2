## 観点別 OK/NG

### 1. 設定ボタンの表示・ダイアログ起動 — OK
左パレットのボタン列(＋/－/名前変更)に「設定」ボタン(AutomationId=SheetSettingsButton)が
表示され、クリックでSheetSettingsDialogが正しく開く。初期値は既定10行・左母線名N24・右母線名
P24(GuiEcad踏襲のデフォルト)。

### 2. Rows/Bus名の編集と保存反映(空文字許容) — OK
Rows=25・左母線名=""(空文字)・右母線名="MyBus"へ変更しOK→再度ダイアログを開いて実値を
確認したところ、いずれも正しく反映されていた(データ側で確認、UI Automation越しの見た目だけに
頼らない)。空文字の左母線名も拒否されず正常に保存された(殿裁定どおりバリデーションなし)。

### 3. 範囲外Rows入力の拒否 — OK
Rows=0・Rows=61のいずれも、OKボタン押下時に赤字で「1〜60の範囲で入力してください」と表示され
ダイアログが閉じない(拒否)ことを確認。スクリーンショットでも表示内容の矛盾なし。

### 4. Rows縮小時のSelectedCellクランプ — OK
5行構成で最終行(行5)を選択中に3行へ縮小 → 選択セルが「行3」(新末尾)へ正しく移動(列は維持)。
視覚的にも選択ハイライトがグリッド範囲内に収まっていた。

### 5. 縮小範囲内要素での拒否+正しい行番号表示(往復2周目の要) — OK
6行構成の行3(縮小範囲の先頭寄り、末尾ではない位置)にコイルM3を配置した状態で2行への縮小を
試行 → ステータスバーに**「行3に要素があるため削除できません」**と、往復1周目以前の固定文言
(「最終行に要素があるため削除できません」)ではなく**実際の該当行番号**が表示されることを確認。
行数も6のまま変更されず(拒否が正しく機能)。往復2周目の修正が実機でも意図通り動作している。

### 6. 既存操作の回帰 — OK
- コイル削除→増分1の「行を追加」ボタンをInvoke→行数が正しく増加し、直前の拒否警告メッセージも
  正しくクリアされた(T-055増分1往復2周目の修正との整合性も保たれている)。
- シート改名(RegressionCheckへ変更)も位置・内容とも正常に反映され、回帰なし。

## 検証手法上の副次的知見

ダイアログ(SheetSettingsDialog等、メインウィンドウとは別のトップレベルウィンドウ)は、
`Save-Ecad2Screenshot`(メインウィンドウハンドル固定のPrintWindow)では内容が写らない
(メインウィンドウの内容のみキャプチャされる)。ダイアログの見た目を確認する場合は、
`AutomationElement.Current.NativeWindowHandle`でダイアログ自身のハンドルを取得し、
そのハンドルに対して直接PrintWindowする必要がある(本検証で回避策を確立、実例は本報告作成の
過程で使用したPowerShellスニペット参照)。スキル側への恒久対応(ヘルパー関数化)は別途検討の
余地あり。
