# T-058 PoC実機確認記録（忍者、2026-07-14）

家老采配（コミット`26b58e6`、`poc/t058-avalondock-poc/`）で侍のUIA自動化では確認不可だった2点の
実機確認記録。DoD：2点の目視確認結果（再現有無）を報告。

## 観点1: タブのマウスドラッグ切り離し→独立ウィンドウ化

**判定: OK（機能する）**

侍のUIA自動化ではSendInput合成イベントがWPFの`DoDragDrop`検知に引っかからず確認不可だったとの
由だが、忍者は`mouse_event`API（SendInputとは別API）+ 多段階ステップ（30ステップ、各50ms間隔で
座標を漸次移動、DOWN→複数MOVE→UP）でのドラッグ合成を試したところ、**切り離しが成立した**。

- 操作: 「ツールパレット(F5-F10相当)」タブのタイトル部分（座標はUI Automationで取得）を掴み、
  右下方向へ大きく（300px, 500px）ドラッグ
- 結果: メインウィンドウから当該パネルが消失、同一プロセスの新規可視ウィンドウ（タイトルなし、
  ドラッグ先座標と一致）が出現。`EnumWindows`で検出し`PrintWindow`で撮影したところ、正真正銘の
  独立ウィンドウ（タイトルバー「ツールパレット(F5-...」、最大化・閉じるボタン付き、中身の
  選択ツール〜部品ボタン全て正常表示）と確認
- 再ドッキングも確認：「ツールパレットを再ドッキング(検証用)」ボタンでメインウィンドウへ復帰、
  ステータスバー「ツールパレットを再ドッキングしました。」表示。ただし**元位置ではなく直前の
  ペイングループ（右パネルの隣）へタブ統合**される形で復帰——これはPoCコミットメッセージ記載の
  既知仕様（「元位置に戻る保証がない」）と一致、異常ではない

**副次発見（侍・家老への技術共有）**：UIA `SendInput`ではWPFの`DoDragDrop`検知に引っかからないが、
`mouse_event`（`SetCursorPos`+`mouse_event(MOUSEEVENTF_LEFTDOWN/LEFTUP)`）による多段階（30ステップ
程度、各50ms）の座標漸次移動でのドラッグ合成は検知される模様。他タスクでドラッグ操作の自動検証が
必要になった際の代替手段として使える可能性あり（ただし今回はPoC単発確認のため、汎用ヘルパー化は
していない。必要なら侍が本実装時に正式検証すること）。

## 観点2: ThemeMode=Dark設定時、起動直後の白フラッシュ

**判定: 再現なし**

- 自動連写での試行: プロセス起動から`MainWindowHandle`確立まで待機し、確立直後から高速連写
  （2回試行、計140フレーム）。両回とも最速でハンドル確立できたのは起動後**694ms時点**で、
  それ以降は全フレームでダークテーマの背景色（RGB概ね42,52,58）が一定して観測され、白系
  フレーム（RGB各200超）は**0件**
- 694ms以前（ウィンドウ作成〜初期描画の瞬間）はPowerShell経由のポーリングでは原理的に観測不可
  （.NETプロセス起動＋アセンブリロードのコストが下限になっている）ため、自動連写では
  「フラッシュが無い」ことの証明にはならない不在証明の限界あり
- そのため**殿へ目視代行操作を依頼**（ninja.md「殿代行操作の運用」）。殿ご自身がexeを2〜3回
  起動し直して目視確認いただいた結果、**「ちらつきは確認できなかった」**との証言を得た

自動連写（694ms以降フラッシュなし）＋殿目視証言（複数回起動でも確認できず）の両方が一致して
「再現なし」を示している。隠密のWeb調査で挙がっていた既知issueは今回のPoC構成
（`ThemeMode="Dark"`固定、AvalonDock 4.74.1、net10.0-windows）では顕在化しなかったと判断する。

## 観点3: 境界線ドラッグでのペインリサイズ（家老追加采配、AvalonDock内蔵リサイザー）

**判定: OK（機能する）**

侍の自動化ツール（SendInput合成）ではタブ切り離しと同じ理由（マウス操作固有の検知問題）で検証
不可だった境界線ドラッグリサイズ。UI Automation上は`ControlType.Thumb`（`LocalizedControlType`=
「縮小表示」）として現れる。観点1と同じ`mouse_event`多段階ドラッグ合成手法で確認した。

- 操作: ツールパレット/左パレット間の境界Thumb（Bounds初期値`2118,113,6,587`）を掴み、右方向へ
  100px、60ステップ・各30ms間隔でドラッグ
- 結果: Thumb座標が`2118`→`2218`（100px分）へ移動、スクショでもツールパレット幅の明確な拡大を
  視覚確認（タイトル「ツールパレット(F5-F10相当)」が省略なしで1行表示されるようになった等）
- **注記**: 初回試行（60px移動・30ステップ・各50ms）では反映されず、距離を増やし再試行して
  初めて成功した。原因未特定（ドラッグ距離の閾値、タイミングのいずれかと推定）だが、機能自体は
  正常に動作することを実機で確認済みにつき実害なし
