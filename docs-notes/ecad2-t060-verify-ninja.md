# T-060（PDF出力UI結線）実機検証記録（忍者）

- 対象コミット: `44d1250`（隠密静的レビュー指摘5件修正、往復1周目、push済み）
- 検証環境: `Start-Ecad2App`（セカンダリモニタ）、PDFはPyMuPDFで実ページ数・表題欄を実測確認

## 結論（観点別、DoD(1)〜(8)全てOK）

### (1) メニュー/ツールバー「PDF出力」押下・Ctrl+Pでプレビューダイアログが開く — OK

ツールバー「PDF出力 (Ctrl+P)」ボタン押下で「PDF プレビュー」ダイアログが正しく開くことを確認。

### (2) プレビューにシートページ・クロスリファレンスページ・BOMページが含まれ、ページ送り・ズーム(0.25刻み)・「幅に合わせる」ボタンが機能する — OK

- 機器(a接点X1・コイルX1)配置状態で「1 / 3 ページ」と表示、次ページボタンで1→2→3ページと正しく
  送れることを確認。1ページ目=シート図面(枠付き)、2ページ目=クロスリファレンス表、3ページ目=
  部品表(BOM)であることを画面キャプチャで確認。
- ズームアウトボタンで100%→75%→50%→25%と0.25刻みで正しく変化することを確認。
- 「幅に合わせる」ボタンでズームが100%（ダイアログ幅に対する相対フィット）に戻ることを確認。

### (3) プレビュー表題欄の総ページ数(N/M表示)と、実際にエクスポートしたPDFの総ページ数・表題欄表示が一致する — OK（重点）

- プレビュー表示「1 / 3 ページ」に対し、表題欄（ズーム50%表示・クロップ拡大で確認）にも
  「1 / 3」の表示を確認。
- 実際にPDF出力（`t060verify.pdf`）し、PyMuPDFで開いて実測：物理ページ数3ページ、表題欄1ページ目に
  「1 / 3」の印字を確認。プレビューと実出力が完全に一致（隠密指摘A=WYSIWYG違反は解消）。
- シート2枚（制御回路+主回路）構成でも再検証：プレビュー「1 / 4 ページ」、実出力PDF
  （`t060verify2sheets.pdf`）も4ページで一致を確認。

### (4) プレビュー閲覧中にEnterを押すと安全に閉じる(PDF出力が走らない)。Escでも閉じる — OK

- Enterキー押下でプレビューダイアログが閉じ、「名前を付けて保存」ダイアログは出現しない
  （新規PDFファイルが生成されていないことも確認）ことを確認。GuiEcad原本同様「閉じる」が
  既定ボタンとして機能している（隠密指摘C解消）。
- Escキーでも同様に安全に閉じることを確認。

### (5) A3等の大きい用紙サイズでもプレビューがダイアログ内に収まり、ウィンドウリサイズに追従する — OK

- UIから用紙サイズを変更する手段が存在しないため、`.gcad`ファイルの`settings.paperSize`を
  直接`"a3"`へ編集したドキュメントを開いて検証（UI上の変更経路が無いことは実装仕様、範囲外）。
- A3ドキュメントのプレビューで、用紙全幅（N24〜P24母線）がダイアログ内に収まりはみ出さないことを
  確認（隠密指摘D=はみ出し確定だった問題は解消）。
- ダイアログを500×500へ縮小しても、ページが自動的に幅に合わせて再描画され、はみ出さないことを
  確認（`SizeChanged`によるリサイズ追従を確認）。

### (6) プレビューの「PDF出力」ボタン→保存ダイアログ→保存で、実際に開けるPDFが生成される。全シート分が出力され枠(表題欄)が常に描画されている — OK

- 生成された`t060verify.pdf`・`t060verify2sheets.pdf`はいずれもPyMuPDFで正常に開けることを確認
  （壊れたPDFではない）。
- シート2枚構成で出力した`t060verify2sheets.pdf`は4ページとも画像化して目視確認：1・2ページ目
  （制御回路シート・主回路シート）はいずれも枠（表題欄）が描画され、ページ番号もそれぞれ
  「1 / 4」「2 / 4」と正しく表示。3・4ページ目（CrossRef・BOM）は元々枠なしの仕様（GuiEcad原本
  踏襲、シートページのみ枠対象）。

### (7) 出力失敗時(例: 出力先PDFを別アプリで開いたまま上書き)に、生の英語例外でなく日本語の固定文面が表示される — OK

- 独立プロセスで`t060verify.pdf`を排他ロックした状態で同名上書きを実行。「PDF出力エラー」
  ダイアログに「PDFを出力できませんでした。出力先の権限やディスクの空き容量、ファイルが他の
  アプリで開かれていないかご確認ください。」＋対象パスのみが表示され、`ex.Message`の生の英語
  例外文面は含まれないことを確認（隠密指摘E解消）。エラー後もアプリはクラッシュせず正常動作を
  継続。

### (8) 既存機能への回帰なし(通常の配置・保存等が正常なまま) — OK

検証中に行ったシート追加・a接点/コイル配置・機器名確定・Ctrl+S保存・Ctrl+O読み込み等の一連操作は
いずれも正常に完了し、異常は見られなかった。

## 隠密申し送り事項（ちらつき）への所見

「ダイアログ初回表示時、一瞬小さいスケールで描画されてから正しいスケールへ切り替わるちらつき」に
ついては、実機目視・スクリーンショット確認では明確な違和感は観測されなかった。ただし瞬間的な
描画事象のため、UI Automation越しの静的なスクリーンショットでは原理的に捕捉困難であり、
「発生していない」と断定はできない（不明点として記録）。
