- ファイルパス等の長文字列直接入力が必要な場面は既存の別手段（最近使ったファイル一覧・既存サンプル
  ファイルの選択等）を優先し、SendKeysでの直接パス入力は避ける

## 観点2: 結線操作

### 試行1: 横配線ボタンInvoke失敗→原因調査

- `Invoke-Ecad2Button -Name "自由線(横線)記入 (F9)"` → 例外`認識できないエラーです。`
  （既知パターン: IsEnabled=Falseのボタンをinvokeしようとした場合の症状と一致）
- 選択ツールに戻す・セル選択しても`IsEnabled=False`のまま変化なし
- **コード確認（MainWindow.xaml）で原因判明**: シート種別によるIsEnabledバインディングの仕様
  - `自由線(横線)記入`/`自由線(縦線)記入`/`接続点記入` → `IsEnabled="{Binding IsMainCircuitSheet}"`
    （主回路シート専用）
  - `縦分岐線記入`/`配線分断記入` → `IsEnabled="{Binding IsControlCircuitSheet}"`（制御回路シート専用）
- 現在のシート「シート1」は種別「制御回路」（シート追加ダイアログの既定値のまま作成）のため、
  横線記入ボタンが無効なのは**正しい仕様**、退行ではない。
- 気づき: 「開く/対象操作前にどちらのボタン群が有効か、シート種別を事前確認する必要がある」
  ことが今回のUIA検証で判明——UI操作ミスであり実装バグではない。
- 検証方針変更: 制御回路シート（現行）では「縦分岐線記入」「配線分断記入」を検証し、別途主回路
  シートを追加して「横線記入」「縦線記入」「接続点記入」を検証する。

### 試行2: 縦分岐線記入（制御回路シート「シート1」）

- `Invoke-Ecad2Button -Name "縦分岐線記入 (Shift+F9)"` → `ツール: PlaceConnector`、案内文言
  「上下キーで範囲、左右キー(Shiftでセル中央)で列位置を調整しEnterで確定、Escで取消」
- `Send-Ecad2Keys "{DOWN}"` → `Send-Ecad2Keys "{ENTER}"`（分割送信、いずれも20文字以内）
- スクリーンショット確認（`t062-vertical-connector.png`）: 行8に●（接続点）、そこから下へ縦線が
  正しく描画。ハイライト選択枠あり。
- **判定: OK**

### 試行3: 主回路シート追加→横線・縦線・接続点記入

- `＋`ボタン→シート追加ダイアログで「主回路」ラジオボタンをUIA SelectionItemPattern経由で選択→OK確定
- 主回路シート（シート2）追加後、`自由線(横線)記入`/`自由線(縦線)記入`/`接続点記入`いずれも
  `IsEnabled=True`に変化（仮説通り）
- **横線記入**: セルクリック→ボタンInvoke→`ツール: PlaceLine`（案内: 左右キーで長さ調整）→
  `Send-Ecad2Keys "{RIGHT}{RIGHT}"`→`Send-Ecad2Keys "{ENTER}"`（分割送信）→確定
- **縦線記入**: 別セルクリック→ボタンInvoke→`ツール: PlaceLine`（案内: 上下キーで長さ調整）→
  `Send-Ecad2Keys "{DOWN}{DOWN}"`→`Send-Ecad2Keys "{ENTER}"`→確定
- **接続点記入**: 別セルクリック→ボタンInvoke→即座に配置（キー操作待ちなし）
- スクリーンショット確認（`t062-horizontal-line.png`, `t062-vline-conndot.png`）: 横線・縦線・
  接続点(●)いずれも正しい位置に正しい形状で描画されている
- **判定: OK（観点2: 結線操作、完了）**。横配線・縦配線・縦分岐線・接続点いずれも正常動作、退行なし。
  なお`Send-Ecad2Keys`の複合キー文字列は21文字以上になりやすく20文字ガードに引っかかるため、
  分割送信で対応した（`-Force`は使わず安全側の運用で足りた）。

- 新規作成(Ctrl+N) → 確認ダイアログなしで新規化（保存直後でDirtyフラグクリア済みのため正常）。

## 観点4: Undo/Redo

### 試行1: 要素配置後にUndoボタンが無効→仕様確認

- シート2(主回路)にa接点を新規配置(行6/列5) → 配置成功（`t062-undo-before.png`）
- `元に戻す (Ctrl+Z)`ボタン確認 → **`IsEnabled=False`**（直前に配置操作したにも関わらず無効）
- **コード確認で仕様と判明**（`MainWindowViewModel.cs`1412-1414行目コメント）:
  「Undo/Redo基盤(T-051)。MVP対象範囲はSheetNavigationViewModelのシート追加/削除のみ」
  — **要素配置・削除・デバイス名変更等はUndo対象外が元々の仕様**（MarkDirty()のみ呼ばれる設計、
  T-019時点の未実装コメントが残っているが1412行目のT-051実装が現行仕様）
- **判定: 退行ではない、既存仕様どおり**。家老委譲のDoD文言「配置操作等でCtrl+Z/Y確認」は、
  シート追加削除のUndo/Redoを指すものとして検証方針を修正する。

### 試行2: シート追加のUndo/Redo
