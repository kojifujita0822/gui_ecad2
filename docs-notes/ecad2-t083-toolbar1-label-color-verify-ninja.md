# T-083 範囲内の気づき実機検証記録（ツールバー1段目ラベル文字色のダークモード非対応）

検証日: 2026-07-18　検証者: 忍者

## 采配内容

家老より、隠密の気づき「1段目`ToolBarKeyLabelStyle`（機能名ラベル）にForeground明示指定が
なく`ToolBar.Foreground`継承依存のまま、2段目`PlacementToolBarKeyLabelStyle`は増分6で
継承に頼らず明示指定と判断済みという非対称性」について、1段目でも継承だけで正しく機能するか
画素採取による機械的判定を依頼された。

## 1. 対象・期待値

- 対象スタイル：`ToolBarKeyLabelStyle`（`MainWindow.xaml:59-63`）、Foreground未設定。
  1段目ToolBar本体（`MainWindow.xaml:627-629`）はForeground="{DynamicResource
  ToolBarForegroundBrush}"を明示、TextBlockはこれを継承する設計。
- 期待値：`Theme.Light.xaml`のToolBarForegroundBrush=`#000000`（黒）、`Theme.Dark.xaml`は
  `#FFF0F0F0`（明色）。継承が機能していれば、ダークモードでラベル文字が明色化されるはず。

## 2. 検証手法

単発ピクセル・単純な色一致比較はClearTypeサブピクセルレンダリングの影響で不安定と判断
（極小フォント(9px)ゆえ文字ストロークが1-2px程度しかなく、大半のピクセルが背景との
アンチエイリアシング中間色になる）。**ライト/ダーク切替前後で、ラベル領域内の非背景色
ピクセルが「暗い方向→明るい方向」に反転するかどうか**を比較する方式を採用（絶対値一致より
頑健）。対象は「新規」「保存」の2ラベル、各ボタン領域(46x42px)を4倍拡大画像でも目視確認。

## 3. 検証結果

### 「新規」ラベル

| モード | 背景支配色 | 非背景色（上位、抜粋） |
|---|---|---|
| Light | 240,240,240 (578px) | 140,140,140 / 72,72,72 / 104,63,104 等、**すべて背景より暗い** |
| Dark | 45,45,48 (578px) | 26,26,28 / 13,13,14 / 20,20,38 等、**すべて背景より暗いまま** |

4倍拡大画像目視：ライトモードでは黒文字相当に見えるが、**ダークモードでも文字が暗いまま**
（背景に近い暗さ）で明色化されておらず、視認性が低い。

### 「保存」ラベル（ダークモードのみ確認）

背景45,45,48に対し非背景色は63,63,65／59,59,61等、**依然として暗い方向**（背景よりわずかに
明るい程度で、期待される明色`#FFF0F0F0`系の値は皆無）。拡大画像目視でも文字は暗いグレーの
ままで視認性が低い。

## 4. 判定：NG（範囲内欠陥）

2ラベルとも一貫して、**ダークモード切替後も文字色が暗いまま**（黒文字相当）で、`ToolBar.
Foreground`継承だけでは正しく明色化されないことを実測で確認した。隠密指摘の懸念通り、
1段目`ToolBarKeyLabelStyle`は2段目`PlacementToolBarKeyLabelStyle`同様に**Foreground明示指定
（`{DynamicResource ToolBarForegroundBrush}`）が必要**と判断する。

参考：同じボタン内のアイコン（Path、`ToolBarIconStyle`のStroke）は正しく明色化されており
対比が明確（アイコンは視認できるがラベル文字のみ暗く沈んで見える状態）。

## 5. 範囲

「新規」「保存」の2ラベルで確認、1段目の他ラベル（開く/戻す/再実行/PDF出力/行追加/行削除/
テスト/一時停止）も同一スタイル(`ToolBarKeyLabelStyle`)を共有するため同様の傾向と推定される
（個別確認はしていない、必要なら追加検証可）。
