# T-083増分4 実機検証記録（選択色系の暗背景視認性確認）

検証日: 2026-07-17　検証者: 忍者

## 采配内容

家老より、`LadderCanvas.cs` の選択色系（OrangeRed/DodgerBlue/White）が
`DrawingTheme.Dark` 背景上で視認十分か実機確認する任務を受領。判定は画素採取による
機械的判定（前セッションの目視誤読2回の教訓を踏まえた指示）。

## 検証観点別 結果

### 観点1: ダークモードで要素選択ハイライト・カーソルが判別可能か → OK（実測確認済み）

手順: Ecad2.Appをセカンダリモニタで起動→シート追加→a接点配置→表示メニューよりダークモード
ON→選択ツールで配置済みa接点を選択→スクリーンショット→画素採取。

実測値（`t083-zoubun4-dark-04-selected.png`）:
- キャンバス背景: `#202226`（理論値と完全一致）
- 選択枠(OrangeRed): `#FF4500`（理論値と完全一致、アンチエイリアシングでない純色ピクセルで確認）

判定: 選択セルの枠が暗背景上で明瞭に判別可能。

### 観点2: OrangeRed/DodgerBlue/White各色の暗背景コントラスト

**OrangeRed → OK（実測+理論値）**
WCAGコントラスト比（背景`#202226` vs `#FF4500`）= **約4.63:1**。
非テキストUI要素基準（WCAG 2.1 SC1.4.11、3:1）を十分クリア、通常テキスト基準（4.5:1）もクリア。

**DodgerBlue → 理論値のみ（実機動的再現は打ち切り、根拠は下記「未検証事項」参照）**
WCAGコントラスト比（背景`#202226` vs `#1E90FF`）= **約4.92:1**。OrangeRedと同水準。
UI要素基準・テキスト基準いずれもクリアする計算結果。

**White → 理論値のみ（未実施、根拠は下記）**
WCAGコントラスト比（背景`#202226` vs `#FFFFFF`）= **約15.93:1**。極めて高く懸念なし。

### 観点3: ライトモードでの回帰なし → OK（実測確認済み）

同一セルをライトモードで再選択し画素採取（`t083-zoubun4-light-01-selected.png`）:
- キャンバス背景: `#FFFFFF`
- 選択枠(OrangeRed): `#FF4500`（ダークモードと同一固定値、変化なし）

判定: ライトモードでの選択ハイライト表示に回帰なし。
（参考: 白背景`#FFFFFF`に対するOrangeRedのコントラスト比は理論値で約3.44:1。UI要素基準はクリアだが
これは本増分のスコープ外の既存仕様であり、今回の変更対象ではない。）

## 未検証事項（実施したが未検証）

- **DodgerBlue・Whiteの実機動的再現**：DodgerBlueは「記入中(未確定)」の一時プレビュー状態、
  Whiteは「画像選択時」のリサイズハンドルであり、いずれもUI Automation単発クリックでの
  安定再現を試行したが、下記の理由により打ち切った。
  - 縦コネクタ記入ツール（「縦分岐線記入 (Shift+F9)」）: ボタン押下→キャンバス単発クリックで
    ツールが意図せず「Select」へ遷移し、ドラフト線が確認できなかった（列0クリックのはずが
    行3/列1にセル選択枠が出現するなど、操作意図と結果が一致しない）。
  - 自由線記入ツール（「自由線(横線)記入 (F9)」）: ボタンInvoke自体が`MethodInvocationException`
    （「認識できないエラーです」）で失敗。
  - White(画像リサイズハンドル)は画像挿入機能自体の操作（ファイルダイアログ等を伴う）が
    必要で、今回のスコープ（色視認性確認）に対し操作コストが高いため未実施。
  - 上記いずれも、マウス移動に追従するリアルタイム状態（ドラフトプレビューは終点をマウス
    位置で更新する設計と推定）が絡む可能性があり、`ecad2-ui-automation`スキル0節の
    「グローバル入力（マウス移動）は殿の他作業と衝突しうるため代替できる観点は前者(非占有操作)で
    済ませる」方針にも照らし、これ以上の粘りは見送った。
  - 根拠として採用した理論値は、コード上いずれも**テーマ非依存の固定色**（`Brushes.DodgerBlue`
    `Brushes.White`）であり、OrangeRedと同型の`Pen`/`Brush`描画方式である点から、動的再現が
    できれば同様に理論値どおりの色で描画されると推定される（未実測の断り付き）。

## 総合判定

三観点いずれもNG無し。ただし観点2のDodgerBlue・Whiteは画素採取による実測ではなく理論値
（WCAGコントラスト比計算＋コード上の固定色確認）による判定である旨、明記して報告する。
