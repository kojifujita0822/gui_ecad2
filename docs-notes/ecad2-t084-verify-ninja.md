# T-084（シート削除時の後始末2件）実機確認記録

忍者、2026-07-15。侍実装(0548983)・隠密静的レビュー完了後の実機確認。

## 観点(1) DRC結果削除の案内表示 — OK

- DRC診断が存在する状態でシート削除→ステータスバーに「DRC結果が削除されました。DRC再実行
  してください。」表示、出力パネルのDRC結果一覧も空にクリアされることを確認。
- DRC結果が既にクリア済み（診断なし）の状態でシート削除→当該メッセージは表示されないことを
  確認（誤表示なし）。

## 観点(2) ページ番号欠番の案内表示 — 条件付きNG疑いあり

- **クリーンな状態（削除前に欠番なし）からの検証**：
  - 非末尾シート削除→欠番メッセージ表示（OK、期待通り）。
  - 末尾シート削除→欠番メッセージ**非表示**（OK、期待通り）。
- **既に欠番が存在する状態からの末尾シート削除 — NG疑い**：
  - 再現手順：(a)シート1/2/3(欠番なし)を作成 (b)シート1(非末尾)を削除→欠番(1)発生
    (c)シート2(非末尾)を削除→欠番(1,2)発生・DRC結果クリア (d)新規シート追加(PageNumber4相当、
    末尾) (e)この末尾シートを削除→**期待は「新たな欠番は生じていないので欠番メッセージ非表示」
    だが、実際には表示された**。
  - 推測（実装未確認、実測結果からの推測）：判定ロジックが「今回の削除で新たに欠番が生じたか」
    ではなく「削除後の残存シート全体のPageNumber列に歯抜けがあるか」を見ている可能性がある。
    その場合、一度でも欠番が生じたプロジェクトでは、以後どのシートを削除しても（末尾であっても）
    毎回欠番メッセージが出続けることになる。
  - 家老の確認観点「末尾以外のシート削除で欠番が生じる場合のみ表示（末尾削除は非表示）」との
    整合性は実装ロジック次第。侍への実装確認・要否判断を仰ぎたい。

## 観点(3) 両条件同時該当時の連結表示 — OK

- DRC診断ありのシートを非末尾から削除→「DRC結果が削除されました。DRC再実行してください。
  シート削除によりページ番号に欠番が生じました。」の連結表示を確認。一方が他方を隠す事象なし。

## 範囲外検出（参考情報、対応不要）

- シート削除後も、機器表（右パネル）に削除済みシートに属していた機器名（Y1・Z1等）が残存し
  続ける現象を複数回観測。機器表の再構築がシート削除に追従していない可能性があるが、T-084の
  スコープ外のため詳細未調査。
- シート追加ボタンを短時間に連続Invokeするとシート追加ダイアログが多重に開いた（UI Automation
  操作特有の罠、スキルのトラブルシュート記載の既知事象を実地で確認。実装バグの可能性は低いと
  見立てる）。
- 新規シート追加時、既定のシート名が既存シートと重複する（"シート1"が複数枚生成される場面が
  あった）。PageNumberとは別の命名ロジックの話でDoDスコープ外だが、参考として記録。
