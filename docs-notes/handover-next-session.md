## 3. 試して失敗したアプローチと結果（今回の教訓）

- **AvalonDockのLayoutAnchorable/LayoutContent系は通常のWPF Binding（DataContext継承前提）が
  機能しない（最重要・往復2周の教訓）**：増分2で`Title="{Binding Find.IsVisible, Converter=...}"`
  という素朴なBindingを書いたところ、二重表示は解消したが**完全に空白**という新たな不具合が発生。
  根本原因＝`LayoutElement`は`DependencyObject`直継承（`FrameworkElement`ではない）、かつ
  `LayoutAnchorablePane.Children`は素のCLRプロパティ(`ObservableCollection<T>`、DependencyProperty
  でない)ゆえ`Children.Add()`がWPFのInheritanceContext確立経路を通らない。**対処＝コードビハインド
  で対象プロパティの`PropertyChanged`を購読し、`LayoutAnchorable.Title`等を直接更新する方式**
  （侍が採用、隠密案のElementNameバインドは不採用）。**T-058増分3（右パネル状況依存切替）は
  この罠に真正面から当たる領域ゆえ、着手前に本節を必ず読ませること**。
- **レイアウトリセット機能の初回実装は「核心的な使用場面」を見落とした**：Ctrl+Alt+Rを
  `MainWindow.PreviewKeyDown`のみに実装した結果、**まさに復旧させたい状況＝フロート化した
  パネル自身にフォーカスがある状態**では効かなかった（殿の実機操作で発覚）。`EventManager.
  RegisterClassHandler(typeof(Window), ...)`でWindowクラス全体を対象にする方式へ修正して解決。
  「対処機能を作ったら、まさにその機能が必要になる最悪ケースで実際に効くか」を確認する教訓。
- **Wチェック方式（侍・隠密独立診断）が今回も機能**：増分2のタイトル空白化バグで、侍・隠密が
  独立に原因調査した結果、表現は違えど「AvalonDockのオフツリー構造」という同一結論に到達。
  診断の信頼性を相互に裏付ける形になった（良い実践、継続）。
