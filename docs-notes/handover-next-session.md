## 3. 試して失敗したアプローチと結果（今回の教訓）

1. **T-119往復2周目：「反転したつもり」が実は無反転だった**。`SplineBorder.OnRender`の座標変換で、
   ベジェ曲線の制御点の"形"（2次×2本→3次×1本）だけを変え、肝心の始点・終点`(w,0)→(0,h)`自体は
   変えていなかった（家老がコード精読で発見）。「見た目を変える」ことと「基準点自体を変える」
   ことを混同しやすい典型例（memory: `feedback_geometric_transform_endpoint_oversight`に記録
   済み）。
2. **T-119「タブ文字消失」問題の認識齟齬**：当初「配置ツールタブ選択時、下部のタブ自体の見出し
   文字が消える」と誤解していたが、実際には全く別の要素——上部の`AnchorablePaneTitle`（青い帯、
   ペインタイトルバー）の表示条件がタブによって非対称だった、という別問題だった（殿の2回目の
   スクリーンショットで判明）。3周にわたり微妙にずれた問題設定で往復していたリスクがあった。
   殿の「私のスクショがわかりにくかったらしい」という申告どおり、視覚的な不具合の報告は
   スクリーンショットの精度が命——今後、画像だけでは判断に迷う場合は積極的に「別の角度・
   分かりやすいスクショ」を依頼すること。
3. **T-119文字消失の対処は仮説ベース**（`TargetName`付きSetterのタイミング問題という推測）。
   (1)(2)は解消したが(3)強調色に別症状が残った——根本原因を完全特定しないまま対症療法的に
   直すと、同じ根本原因の別の現れ（Triggerの種類違い）を見落とす典型例になりかけている。次回は
   同じ設計変更（`TemplateBinding`化）を残る箇所（強調色のTrigger）にも横展開することから
   着手し、それでも解消しなければ`karo.md`「モグラ叩き検知」どおりアプローチ自体の俯瞰評価に
   進むこと。
4. **忍者への気づき収集で応答遅延が発生**：侍・隠密は速やかに返信したが、忍者からの返信が長時間
   届かず、家老が催促して初めて届いた（送信自体は忘れていなかったとのこと）。`karo.md`へ
