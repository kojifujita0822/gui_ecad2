## 運用ルール（追加・教訓）

1. **アイドル中のセッションはpeerメッセージを受けても動かない**。返答なき采配は (1)殿にそのターミナルへ一言打ってもらう (2)再起動後はメッセージ队列が消えるため采配を必ず全量再送。
2. **pushは家老経由が既定**（侍はコミットまで。家老がpush前に`git log origin/main..HEAD`で混入確認）。今セッションで侍が直接pushした例あり→注意済み・以後は委ねる旨合意。
3. **peer名簿の亡霊対処**: 名乗らぬpeerが居たら `Get-Process -Id <PID>`＋`Win32_Process`の親チェーンで正体を検める。今セッションで2体処理: (a)`claude.exe.old.*`配下=自動更新前の孤児セッション (b)`claude.exe daemon`→`--bg-pty-host`→`--resume`配下=デーモンの背景復元セッション。いずれも殿の裁可を得て停止（デーモン本体は残す）。
4. 従来ルール継続: 忍者の検証build前`git status --short`確認／peerメッセージ要約／検証パイプライン既定順（侍→隠密静的→忍者実機）／修正往復2周上限／Wチェック並行方式／MouseAssistant競合対処／UI/UX分岐は必ず殿へ。
