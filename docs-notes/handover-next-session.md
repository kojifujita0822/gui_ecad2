## 3. 試して失敗したアプローチと結果（今回の教訓）

T-099(c)は「配置ツールバー上段タイトルのラベル非表示化＋フロート化・再ドッキング」を巡り、
**6段階以上の対処を経てなお解決せず、最終的に方式転換（案Y）へ至った**長期調査だった。

1. **Visibility=Collapsed**（前セッション由来）→フロート化ドラッグ処理自体が死ぬ重大回帰
   （既存教訓、[[wpf_collapse_visibility_hides_interaction]]）。
2. **Opacity=0+Height=1**→ヒットテスト領域が極小でドラッグ開始自体が困難（UIA・物理マウス
   とも不成立）。
3. **Height=5**（殿裁定「動作優先」）→UIA環境では動作するが、**殿の物理マウス操作では
   依然フロート化できず**（隠密調査でAvalonDock`InternalOnActivated`のタイミング問題と特定、
   MouseAssistant競合は否定済み）。
4. **上段テンプレート再設計（MenuDropDownButton活用）**→フロート化自体は成立したが、「ドッキング」
   メニュー選択で**タブが自己複製する重大バグ**（AvalonDock標準の`InternalDock`フォールバック
   探索にフロートウィンドウ除外フィルタ漏れという内部バグ、`ContentDocking`の自前ハンドリングが
   誘発）。
5. **独自再ドッキング処理（CollectGarbage/InvalidateArrange追加）**→タブ複製は解消したが、
   後始末不足でツールバー中身が消失。**UpdateLayout遅延実行→InvalidateVisual追加**と対症療法を
   重ねるも、最終的に**レイアウト永続化ファイル（`%AppData%\Ecad2\docking-layout\
   placement-toolbar.xml`、gitの管理外）が実験で汚染され、起動時から配置ツールバーが完全に
   空白・Ctrl+Alt+Rも無効**という最悪の回帰を招いた。汚染ファイル退避で復旧（削除は殿の明示
   許可を得た一時例外）。
6. 復旧後の再検証でも「縦長44pxタブとして左端に別追加」という別形の異常が再発——**殿より
   「根本的に直して、プランの練り直しも検討して」との明確な指示**を受け、隠密が最深部の一次
   ソース調査（AvalonDock`LayoutRoot`の「RootPanelがnullになると空のDocumentPaneで自動補完
   する」隠れた不変条件）で完全な根本原因を特定。

**教訓（隠密所見、家老がkaro.mdへ制度化済み＝「モグラ叩き検知」節）**：`ContentDocking`イベント
を自前ハンドリングしモデル手術する方式自体に構造的限界があった——AvalonDockの隠れた不変条件
全てと自前で整合を取る必要があり、直すたび新たな不変条件と衝突するモグラ叩きに陥っていた。
**2周目の時点で「介入レイヤー自体が間違っていないか」を俯瞰評価すべきだった**。

その他の個別教訓は memory 化済み（[[feedback_investigation_layer_and_scope]]・
[[feedback_visual_bug_and_oss_source_investigation]]・[[ecad2_ui_automation_pitfalls_t099]]・
[[avalondock_hidden_invariants_survey_entry_points]]）。
