  直後、侍の現行セッションはそれを知らぬまま作業を続けていた（本セッションで発覚・訂正）。以後は
  「該当ファイルパスを明記し`Read`で確認させる」運用を徹底すること（karo.md制度化済み）
  ——**家老の要約を書き写すだけでは不十分**、当人に原文を読ませる
  一手間を必ず入れる

  重ねたことで、侍のローカルHEADのみ家老の直近コミットを意図せず上書き（origin/mainは無傷）。
  `git reset --hard origin/main`で復旧（家老が安全性を二重確認の上許可）。**同一working tree
  を複数セッションが共有する構造ゆえ、家老・侍が同時にgit操作を行うタイミングには要注意**
- **T-058のバージョン前提は状況が変わった**（前回引き継ぎからの継続事項）：「4.74.1で開始」裁定は
  ecad2がnet8だった頃の判断。T-062完了（.NET 10）により5.0.0系（net10明示対応）直接採用の選択肢が
