## 技術教訓（本日分）

1. **FocusScope跨ぎではUpdateSourceTrigger=LostFocus（論理フォーカス）が物理操作でも発火しない**。パネル分割+FocusScope構成のこのアプリでは、TextBox系の確定契機は**LostKeyboardFocus+Enter明示のUpdateSource（Explicit化）**を標準とすべし（T-036の確定パターン、DeviceNameBoxで実証済み）。
2. **LostKeyboardFocus契機はUIA検証とも相性が良い**（忍者が3経路一発検証に成功。LostFocus時代はUIA困難だった）。
3. **T-038運用（診断ログ連携）の初適用が成功**: 忍者3パターン打ち切り→両論併記報告→侍ログ注入→殿実操作→隠密理論と突合、の流れで構造バグを短時間で確定。
4. **状態リセットの単一責務点**: ReplaceDocumentに「文書に紐づく状態」のリセットを集約（前任侍の勘所、handover前版から継承。詳細は台帳T-019備考）。
5. **CurrentSheetIndex既定値0の意味論重複**（「シート無し」vs「先頭選択中」）は通知漏れの温床。番兵値化(-1)は見送り中、シート操作系を触る際は注意（同上）。
