## 3. 試して失敗したアプローチと結果（今回の教訓）

- **BashツールでPowerShellの日本語ヘルパースクリプトをdot-sourceすると文字化けでパースエラーに
  なる**（`ecad2-ui-automation`スキルのhelpers.ps1で実際に発生）。実機確認でPowerShellスクリプトを
  呼ぶ際は必ず**PowerShellツール**を使うこと、Bashツールは不可。
- **`Get-Ecad2Process`は`-ErrorAction SilentlyContinue`のため例外を投げない**——
  `try { Get-Ecad2Process | Out-Null; 'RUNNING' } catch {...}`という判定コードは、プロセスの
  有無に関わらず常に'RUNNING'と誤判定する。正しくは`if ($p) {...}`のように戻り値の有無で判定する
  こと（今回、この誤判定により忍者の「アプリは起動していない」という正しい報告と一瞬矛盾が生じた）。
- **Escチャタリング／キー判定過敏という代替仮説は、ログ上「Escape到達記録が皆無」という事実で
  否定はできないが完全排除もできない**（殿から留保あり）。ただしチャタリングで2回Escが送られた
  としても、1回目でContextMenuが閉じれば2回目はメインウィンドウに届くはずで、それでも記録が
  皆無だったことから「メインウィンドウはEscapeを一度も受け取っていない」という結論自体は3回とも
  再現し崩れない、と説明し殿の了解を得た。今後同様の懸念が出たら、この論法を踏まえた上で改めて
  調査に応じる旨、殿へ伝達済み。
