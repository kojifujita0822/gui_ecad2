## 3. 試して失敗したアプローチと結果(今回の教訓)

- **出力破損がコミットメッセージ生成テキストに2回連続混入した**(#11「実機確認(忍者)」→
  「実機確認(�містниくindex)」、#12「掴んだハンドル」→「�015んだハンドル」、いずれもgit heredoc
  本文中)。両者ともRead/Grep経由ではなく、自分自身のテキスト生成過程での混入という新型
  (docs-notes/output-corruption-log.md #11, #12参照)。
- **`git commit --amend`は並行作業セッションがある環境では危険**：#11修正時は問題なかったが、
  #12修正時にamendを実行した際、その間に家老が2件コミットを積んでいたのに気づかず、HEAD
  (=家老の直近コミット`a6afa10`)を誤って上書きしてしまった。origin/mainは家老が既にpush済みで
  無傷だったため`git reset --hard origin/main`(家老許可済み)で実害なく復旧できたが、
  **教訓: コミットメッセージ破損の修正はamendでなく新規コミットで訂正する方が安全**
  (今回の実装コミット5a2568a自体はamendせず、記録簿だけ別コミットにすべきだった)。
- 4セッション体制では他役(特に家老)が並行してコミット・pushを行いうるため、`git log --oneline -3`等で
  直近のHEADを確認してからamend系操作を行う習慣が必要。
