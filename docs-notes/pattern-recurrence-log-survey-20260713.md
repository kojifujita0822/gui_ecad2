# 過去レビュー書棚卸し 調査記録(2026-07-13、隠密2)

殿ご指示(2026-07-13、引き継ぎメモ経由)により、`docs/archive/`配下のファイル名に"review"を含むレビュー書81件を
9班に分けサブエージェントへ速読調査を委譲、`docs-notes/pattern-recurrence-log.md`(不具合パターン再発台帳)へ
未記帳の同型再発パターンを洗い出した記録。

## 対象範囲

`docs/archive/*review*.md` 全81件(find実測で確認、Globツールの初回応答は表示乱れがあったが実ファイル数と一致)。
9班(各9件)へ均等分割し、各班に「要修正/CONFIRMEDのみ抽出・根本原因タグ付け・自班内での2件以上再登場を自己申告」を依頼。

## 統合分析(タスクID単位での重複判定)

各班は自班9件内でのみ重複を検知できるため、隠密2にて**班をまたぐ突合**とタスクID単位での集計を行った。
「同一タスクの往復レビュー内での再確認」(例: T-082初回→往復2周目で同一バグの再確認)は独立事例として
数えず、**異なるタスクIDにまたがるもの**のみを「複数タスクにまたがる型」と判定した(家老指示に基づく基準)。

### 既存パターン(PR-01/PR-02)の制度化前・遡及実例

台帳のPR-01/PR-02は2026-07-12〜13に初出記帳されたばかりだが、今回の棚卸しでそれ以前の同型実例が
多数見つかった。制度化はいずれも直近だが、根本原因自体はプロジェクト初期から繰り返されていたことを示す。

**PR-01(新規選択可能状態の横展開漏れ)の遡及実例**:
- T-019(d9aa49b): ウィンドウクローズ操作(×ボタン・Alt+F4)に`ConfirmDiscardIfDirty()`が未組込
- T-033(増分1): 非モーダル配置バー表示中、マウス経由6系統(キャンバス・部品選択リスト・DRC出力パネル等)にガードが素通し
- T-041増分1: `SelectedConnector`クリアが4箇所(矢印キー・選択ツールボタン・DRC同一シートジャンプ・新規/開く)で漏れ
- T-041増分2: `_connectorDraft`/`Tool.Mode`がシート切替時にクリアされない
- T-041増分7: `_draggingConnector`が既存クロスカット選択クリアパターンの外にあり3経路でリーク
- T-043→T-045増分B: Id完全一致判定の弱点修正がRenderer側のみで、Converter側・T-045増分Bの`ResolveDeviceClass`に未反映のまま再発(**同一パターンが2タスクにまたがって未修正のまま連鎖**という点で特に根深い実例)
- T-051: `ReplaceDocument`がUndoManagerクリア責務を持たず、`ApplyUndoRedoSnapshot`もSelectedSheet通知・OutputPanelクリアを欠く
- T-055増分1: `DeleteRowCommand`が行削除後のSelectedCellクリア・再クランプを欠く
- T-080: 行コメント編集状態がIsEnabledバインドに未反映
- T-082初回: `MoveSheetCommand`だけAdd/Delete/Renameが持つSelectedSheet通知規約を踏んでいない

**PR-02(未確定編集の確定漏れ)の遡及実例**:
- T-051: Ctrl+Z/Yに`CommitDeviceNameEdit()`ガードが無い
- T-063: `DeleteMenuItem_Click`が`CommitDeviceNameEdit()`を呼ばない
- T-065・T-066: 型式セル編集の確定漏れ(文書自身が「T-049/P-013の再発」と明記)

(T-036の`CommitDeviceNameEdit`類似指摘は、レビュー自身が「T-036の新規回帰でなくT-019由来の既存課題」と
判定しているため、独立実例としては数えない)

## 新規パターン候補(PR-01/PR-02以外)

### 複数タスクにまたがり正式パターン相当と判断したもの(家老への強調対象)

**PR-03候補: 早期return(SetPropertyの値不変判定によるクリア処理スキップ)**
- 型: 参照先(実体)は変わったが、SetPropertyが比較する値(index等の数値キー)が偶然一致し、
  本来発火すべき`OnPropertyChanged`ひいては連鎖するクリア・再描画処理が丸ごとスキップされる
- 該当タスク: T-019(b7844d2、起動直後Sheets=0→最初のシート追加でCurrentSheetIndex=0のまま)、
  T-041増分5(シート削除でCurrentSheetIndex数値不変)、T-082(MoveSheetCommandで添字変化のみ判定)
- 3タスクにまたがる。既存メモリ`ecad2_setproperty_early_return_trap.md`が指す罠と直結するが、
  台帳(pattern-recurrence-log.md)には未記帳だった

**PR-04候補: 境界値/クランプ検証漏れ(新規プリミティブ・座標境界チェックの個別実装分散)**
- 型: 新しい図形要素・操作を追加する際、既存要素で確立済みの境界クランプ・範囲検証(Grid行数、
  画面外周、min>maxガード等)がその新規実装には移植されず、クラッシュや視覚異常を招く
- 該当タスク: T-033(増分2、クランプの非重なり非保証)、T-041増分7(VerticalConnector Top/Bottom未検証、
  FreeLineクランプ欠如、ConnectionDot未適用)、T-051(round3、SelectedCell復元のGrid.Rows範囲クランプ漏れ)、
  T-055増分1(DeleteRowCommandのクランプ漏れ)
- 4タスクにまたがる、最も広く再発している型の一つ

**PR-05候補: 状態リセット処理の横展開漏れ(Document/Sheet構成変更時)**
- 型: PR-01の親戚筋。文書・シート構成を差し替える処理を新設する際、既存の同種処理
  (`ReplaceDocument`/`SetCurrentSheetIndexCore`等)が担う状態クリア責務(UndoManager・OutputPanel・
  SelectedSheet通知・SelectedCellクランプ)への追従が漏れる
- 該当タスク: T-051、T-055増分1
- 2タスク。PR-01とスコープが重なるため、統合するか独立記帳するかは家老判断を仰ぎたい

**PR-06候補: 型強制不足(規約レベルの誘導止まり)**
- 型: 「正しい呼び出し順序・対の処理」が型システム(コンパイラ)で強制されず、コメント・テスト・
  レビューという人的手段のみに依存。将来の拡張(第5のドラッグ型追加等)で同型欠陥が再発しうる
- 該当タスク: T-045増分D、T-050(fix2、SetCurrentSheetIndexCoreのペアリング不変条件)
- 2タスク

**PR-07候補: コピペ重複/共通ロジックの複数箇所分散(rule of three到達)**
- 型: 同一ロジック(SelectedCellクランプ式・StatusMessage/MarkDirty/Notifyトリプレット・
  機器表クリーンアップ・ページ構成ロジック等)が3箇所目の複製に到達、過去に実際の書き漏れバグ実績あり
- 該当タスク: T-055(増分2・増分3差し込み分)、T-060(プレビュー/実出力PDFのページ構成ロジック二重実装)
- 2タスク

### 単発(1タスクのみ、台帳に「新規パターン候補」として記帳のみ)

- **SetPropertyバイパスによる旧値ロスト**: T-039のみ。SetProperty非経由の直接代入+1引数版通知で
  旧値が常にnullになる。侍が個別箇所を直すたび兄弟コード(`SaveToFile`→`DeviceTableViewModel`等)に
  同型が新たに見つかる「もぐら叩き」型の再発を同一タスク内で3回確認
- **文字列正規化漏れ**: T-039のみ。半角"0"→全角０→全角ラテン文字と、直すたび対象範囲が拡大
- **旧値捕捉タイミング誤り**: T-050のみ。`oldValue`をコレクション/インデックス変更後に読んでしまい
  old==newや無関係な値になる。SetPropertyバイパスとは別メカニズムだが「PropertyChangedのoldValueが
  不正確」という症状面では上記と関連しうる
- **座標系(絶対/相対)取り違え**: T-033のみ(`PositionPlacementBar`の初回バグと修正後に残った同型の
  脆弱性)
