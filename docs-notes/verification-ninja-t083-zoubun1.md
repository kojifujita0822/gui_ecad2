# T-083増分1（AvalonDockテーマ連動、層B）実機検証記録

検証者: 忍者（key=1784187128338） / 検証日: 2026-07-16
対象コミット: 0f63245 + 5020811 + 21d58fb（`src/Ecad2.App/`配下）

## 結論（先出し）

**観点2・観点4がNG**。ダークモードトグルをON/OFFしても、4つのDockingManager
（配置ツールバー・シート・機器表/プロパティ・出力）のドッキングクローム（タブ・タイトルバー）が
VS2013 Dark/Lightへ一切切り替わっていない。ドッキング状態・フロート状態のいずれでも同様。
プロパティパネルの中身の背景色のみ暗色化しているが、これはドッキングクローム自体ではなく
別経路（層A/層C系統？）の影響と見られる。

一方、既存PoC（層A＝作図キャンバス色）は正しく連動しており回帰なし。

## 観点別 OK/NG

### 観点1：起動直後（トグル未操作）のVS2013 Light既定適用 — OK

起動直後、4パネルとも白背景・青系タブヘッダーのLight相当の見た目。Aero系既定のまま取り残された
様子はなし。（`t083-1-initial.png`）

### 観点2：ダークモードトグルON時、4つ全てがVS2013 Darkへ即時切替 — **NG**

トグルON後も配置ツール・シート・機器表・出力のタイトルバー（ドッキングクローム）は白/明るい青の
まま変化なし。プロパティパネルのみ中身の背景色が暗色化するが、タイトルバー自体はLightのまま
（部分反映）。シート追加でアクティブになったシートパネルのタイトルバーも、Dark配色ではなく
通常の明るい青（Light時と同じ配色）だった。
（`t083-1-darkon.png`, `t083-1-darkon-retry.png`＝再現性確認のため1秒待って再撮影、同一結果。
`t083-1-sheet-added-clean-darkon.png`＝シート実在状態での撮影）

### 観点3：OFF時、正しくLightへ戻る — OK（ただし観点2がNGにつき実質的意味は限定的）

OFF操作でプロパティパネルの中身は白へ戻ることを確認。ドッキングクロームは元々Lightのまま変化
していなかったため「戻る」という程の変化はない。（`t083-1-sheet-added-clean-darkoff.png`）

### 観点4：フロート状態でのテーマ反映 — **NG**（観点2と同型）

シートパネルをドラッグして切り離し、フロートウィンドウとして独立させた状態で確認。フロート後の
タイトルバーもドッキング時と同じく明るい青（Lightスタイル）のまま、Dark配色になっていない。
ドッキング/フロートを問わず一貫してテーマ未反映という所見。（`t083-1-float-clean-darkon.png`）

### 観点5：既存PoC（層A・キャンバス色）の回帰確認 — OK

シートを実在させた状態で確認したところ、キャンバスはLight時＝白背景・グレーグリッド線、
Dark時＝黒背景・グレーグリッド線と正しく連動して切り替わっていた。回帰なし。
（`t083-1-sheet-added-clean-darkon.png` / `t083-1-sheet-added-clean-darkoff.png`）

**注記**：検証序盤（シート0件の状態）ではLight/Dark両方でキャンバスが同じ濃紺色に見え回帰を
疑ったが、これは既知の範囲外事項（PoC時点で報告済み「シートが0件の時のみキャンバスが固定の
濃紺色になる」）に該当するだけで、シートを実在させた状態では正しく連動していることを確認済み。

## 検証中のトラブルと対処（手順の教訓）

観点4着手の1回目の試行で、直前に押した「＋」（シート追加）ボタンが実はモーダルダイアログ
「シート追加」を開いていたことに気づかず、ダイアログが背後に開いたまま表示メニュー操作
（ExpandCollapse・トグルInvoke）を続けてしまった（スキル既知トラップ「モーダルダイアログを
開くボタンのInvoke後はEnumWindowsで多重化・残存確認必須」に該当する事例）。EnumWindowsで
2枚目の可視ウィンドウ（Size=320x210）を検出して発覚、キャンセルボタンをInvokeしてクリーンな
状態に復帰させた上で、シート追加→OK確定→観点4再試行という正しい手順でやり直した。
なお、この手順ミスの影響下にあった観点2の所見自体は、手順ミス以前（シート追加前、クリーンな
状態）での観察と一致しており、結論への影響はない。
