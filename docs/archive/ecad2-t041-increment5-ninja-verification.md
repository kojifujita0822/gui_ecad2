## 範囲外検出

1. **F10（WireBreak）配置直後、キャンバス再描画が一時的に反映されないケースを観測**。制御回路
   シートの空セル（配線なし）でF10を押した際は正しく無反応（対象なしのため何も生成されない）
   だったが、配線がある位置でF10を押した直後のスクリーンショットでは要素が見えず、その後の
   別操作（セルクリック等）を挟んだ後のスクリーンショットで初めてWireBreak（赤丸）が表示される
   事象を1回観測した。Undoで取り消せたことから、配置自体は成立していたと判断する。再現条件を
   完全には切り分けられておらず、UI Automation操作特有のタイミング問題（ninja.md既知の罠）の
   可能性もあるため、断定はしない。家老・侍の判断を仰ぎたい。
2. **シート追加・削除操作がCtrl+Z（Undo）の対象外に見える**。誤操作で主回路シートを削除した後
   Ctrl+Zを送ったが復元されなかった（要素の配置・削除はUndo可能なことを他の場面で確認済み）。
   意図した設計か未確認のため、気づきとして申し送る。

いずれも今回のDoD（増分5クローズ判定）には影響しないと判断する。

## 検証時の注意点（引き継ぎ、ninja.md追記候補）

UI Automationの`Invoke-Ecad2Button`でボタンを押した直後、フォーカスがそのボタン要素に残留する
ことがあり、この状態で`Send-Ecad2Keys "{ENTER}"`を送るとキャンバスではなくフォーカス保持中の
ボタンが再度アクティベートされる事象を2回観測した（「－」シート削除ボタン→意図せずシートを
二重削除、「名前変更」ボタン→改名ダイアログが再オープン）。両方とも実害なく気づいて復旧できたが、
ボタンInvoke後にキーボード操作（Enter等）を送る場合は、間にキャンバスへのクリックを挟む、または
`Find-Ecad2Element -AutomationId "CanvasArea"`から取得した要素で状態を確認するなど、フォーカス
先を意識した方がよい。
