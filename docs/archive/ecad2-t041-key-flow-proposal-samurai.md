# T-041 記入キー操作フロー原案 — 2026-07-07（侍）

家老采配「増分2着手前にキー操作フロー原案を提示せよ」への回答。**本段は原案提示のみ、実装は
まだ行わぬ。** UI/UX分岐を含むため、家老経由で殿の裁定を仰ぎたい。

対象：初版プラン2節で示した「線系（`VerticalConnector`/`FreeLine`）」「点系
（`WireBreak`/`ConnectionDot`）」の構造対称性を踏まえ、4種すべてのキー割当・操作フローを
まとめて提示する（実装順は既定どおり増分2〜5だが、キー割当は4種で矛盾なく確定させたい）。

## 1. 前提の整理

`docs/todo.md`のT-041行は「主回路用の横配線・縦分岐線」（＝主回路シートで横方向/縦方向の
`FreeLine`）を指しており、F9/sF9はもともとこの2方向のための2キーだったと解釈できる。本日の
殿裁定でスコープに加わった`WireBreak`（制御回路・点）・`ConnectionDot`（主回路・点）を同じ
F9/sF9に載せるか、専用キーを新設するかが未確定の論点。

- 制御回路シート（`MainCircuit==false`）：`VerticalConnector`（線）＋`WireBreak`（点）
- 主回路シート（`MainCircuit==true`）：`FreeLine`（線、横/縦の2方向）＋`ConnectionDot`（点）

## 2. キー割当案（2案、殿確認事項）

### 案A：線系＝F9/sF9、点系＝専用キー新設（侍推奨）

- **制御回路シート**：sF9＝縦分岐線（`VerticalConnector`）記入モード開始。F9＝当面未使用
  （制御回路の横配線は自動計算のため対応する`FreeLine`的な横線記入は存在しない）。
- **主回路シート**：F9＝横自由線記入モード開始、sF9＝縦自由線記入モード開始。
- **点系専用キー（新設、両シート共通）**：例＝**F10**。制御回路シートでは`WireBreak`、
  主回路シートでは`ConnectionDot`を記入（対象データ構造はF9/sF9と同様シート種別で自動切替）。
  押下と同時に即時記入（3節参照、確認フェーズ不要）。

長所：各キーの役割が固定的で発見しやすい（GX Works3等の「1キー1機能」規約に近い）。
短所：新規キー（F10）が1つ増える。

### 案B：記入モードの入口を共有し、種別は後続の最初のキー入力で分岐

- 制御回路シート：sF9＝「配線記入モード」開始。直後に矢印キー（上下）を押せば縦分岐線
  （線系）、Enterを即押せばその場に配線分断（点系）を記入、という分岐。
- 主回路シート：F9＝「配線記入モード」開始。直後に上下矢印＝縦自由線、左右矢印＝横自由線、
  Enter即押し＝接続点、という3分岐。
- 長所：新規キーを増やさずF9/sF9のみで4種すべてをカバーできる。
- 短所：「モードに入っただけでは何を記入するか分からない」ため発見しやすさ（discoverability）
  に劣る。ステータスバー案内文（例：「矢印キーで線、Enterで点」）で補う必要がある。

**侍としては案A（専用キー新設）を推奨する**：役割が固定的でユーザーが覚えやすく、既存の
F5〜F8（1キー1種別）の規約とも一貫する。ただし新規キーが1つ増える点は殿の判断を要する。

## 3. 線系（`VerticalConnector`/`FreeLine`）の記入フロー具体形

前提：`SelectedCell`が選択済みであること（未選択時はF5-F8のTryPlaceBuiltinと同様、
ステータスバーに「配置するセルを先に選択してください」を表示し記入モードへ入らない）。

1. **開始**：sF9（制御回路）またはF9/sF9（主回路、方向で分岐）を押すと、`ToolMode`が新設の
   記入モード（既存`ToolState.PlaceConnector`/`PlaceLine`の列挙値を流用）へ遷移する。
   始点は`SelectedCell`から導出する：
   - `VerticalConnector`：始点行＝`SelectedCell.Row`、始点列境界＝`SelectedCell.Column`
     （セル左端。既存のOR自動配線の縦コネクタ列決定ロジックと単位を揃える）。
   - `FreeLine`：始点は`SelectedCell`をmm座標へ変換した位置（`GridGeometry.X`/`YRow`）。
2. **調整（矢印キー、リアルタイムプレビュー）**：
   - `VerticalConnector`：Up/Downで終点行を1行ずつ伸縮、Left/Rightで列境界を0.5刻みで移動
     （Shiftなしは整数境界、Shift+Left/Rightでセル中央=X.5境界、既存OR自動配線の境界値と
     同じ刻み方に揃える）。
   - `FreeLine`（横）：Left/Rightで終点のみ移動（Y座標は始点と同一に固定、水平線を保証）。
   - `FreeLine`（縦）：Up/Downで終点のみ移動（X座標は始点と同一に固定）。
   - 押下のたびに`LadderCanvas`へ「記入中プレビュー」を仮想`VerticalConnector`/`FreeLine`
     として渡し、増分1で作った選択ハイライトと同系統の描画（点線等、確定前と分かる見た目）で
     即時反映する。
3. **確定（Enter）**：現在のプレビュー形状で`sheet.Connectors`/`sheet.FreeLines`へ`Add`し
   `MarkDirty()`。`ToolMode`を`Select`へ戻し、`SelectedCell`は保持（連続記入がしやすいよう
   T-021分岐Aの「ツール保持」方針に倣う。ただし記入モード自体は1回で終了し、続けて記入するには
   再度sF9/F9を押す）。
4. **取消（Esc）**：既存のEsc層2（配置モード中→選択モードへ戻す）と同じ扱いで、プレビューを
   破棄し`ToolMode`を`Select`へ戻す。要素は生成しない。

## 4. 点系（`WireBreak`/`ConnectionDot`）の記入フロー具体形

線系より単純：確認フェーズを設けず、キー押下と同時に即時記入する（案A採用時、F10相当のキー）。

1. `SelectedCell`が選択済みであることが前提（線系と同様）。
2. 押下と同時に、`SelectedCell`から導出した位置へ即座に`Add`＋`MarkDirty()`：
   - `WireBreak`：`Boundary`＝`SelectedCell.Column`のセル中央（X.5）、`Row`＝`SelectedCell.Row`。
   - `ConnectionDot`：`SelectedCell`をmm座標変換した位置。
3. 取消操作は無い（記入後に不要なら増分1と同じ「選択→Delete」で消す。GuiEcadの`WireBreak`
   トグル方式は不採用、既定方針どおり全4種で「選択→Delete」に統一）。

## 5. 未解決の論点（殿確認を要する、本原案の範囲外）

- 「修正」（増分6/7、ドラッグ＋キーボード等価操作の案X）の具体的なキー割当は本原案では扱わない
  （別途増分6のPoC結果を踏まえて提示する）。
- 案Bを採る場合、`FreeLine`の斜め線（水平・垂直以外の任意角度）は本フローでは表現できない
  （矢印キーは軸方向にしか対応しないため）。GuiEcadのマウスドラッグは任意角度に対応するが、
  キーボード操作を主手段とする以上、**斜め線はスコープ外（水平・垂直のみ）としてよいか**も
  併せて確認したい。
