# T-048 実機回帰確認：手動配線sF9/F10系グリフのシート種別描き分け

検証日: 2026-07-10（忍者 key=1783637541231）
対象コミット: `0986c81`（`src/Ecad2.App/MainWindow.xaml`のみ、Path Data 2箇所変更。
隠密静的レビュークリーン＝Trigger類無傷・便乗変更なし）

見た目自体（矢印形・はさみ形）は殿プレビュー承認済みのため、本検証は**回帰確認に限定**した。

## 検証方法

`ecad2-ui-automation`スキル使用。`dotnet build`0警告0エラー確認後起動。
判定はUI Automationの`IsEnabled`プロパティ（データ）+スクリーンショット（見た目）の両輪。

## 観点別判定

### 観点1: 制御シートでの活性/グレーアウト（判定：**OK**）

制御回路シート作成後、F9系・F10系5ボタンの`IsEnabled`を確認：

| ボタン | IsEnabled |
|---|---|
| 自由線(横線)記入 (F9) | False（グレーアウト、主回路専用） |
| 自由線(縦線)記入 (Shift+F9) | False（グレーアウト、主回路専用） |
| **縦分岐線記入 (Shift+F9)（矢印形・新グリフ）** | **True（活性）** |
| 接続点記入 (F10) | False（グレーアウト、主回路専用） |
| **配線分断記入 (F10)（はさみ形・新グリフ）** | **True（活性）** |

スクリーンショット（`t048-02-controlsheet.png`）でも矢印形・はさみ形が黒地で活性表示、
他3ボタンが半透明グレーで表示されていることを確認。T-047確立の半透明化（Opacity 0.35系）
挙動と見た目上の整合を確認。

### 観点2: 主回路シートでの活性/グレーアウト（判定：**OK**、観点1と完全に逆転）

新規シート追加時に「主回路」を選択して作成、同5ボタンを確認：

| ボタン | IsEnabled |
|---|---|
| 自由線(横線)記入 (F9) | True（活性） |
| 自由線(縦線)記入 (Shift+F9) | True（活性） |
| 縦分岐線記入 (Shift+F9)（矢印形） | False（グレーアウト、制御回路専用） |
| 接続点記入 (F10) | True（活性） |
| 配線分断記入 (F10)（はさみ形） | False（グレーアウト、制御回路専用） |

観点1とシート種別切替で正しく入れ替わることを確認（`t048-08-maincircuit.png`）。

### 観点3: ボタン並び順（判定：**OK**、T-047確定時と不変）

ツールバー全ボタンを左端X座標順に並べ、`選択(Esc)→F5→sF5→F6→sF6→F7→F8→
自由線横線(F9)→自由線縦線(sF9)→縦分岐線(sF9)→接続点(F10)→配線分断(F10)→自作パーツ`の
順であることを確認。T-047確定時の並び「選択→F5〜F10→区切り→部品」と一致。

### 観点4: グリフ変更ボタンの機能起動（判定：**OK**）

制御回路シートで新グリフ2ボタンを実際にクリックし、機能が従来どおり起動することを確認：

- **縦分岐線（矢印形）**：クリックでツール状態が`Select`→`PlaceConnector`へ遷移、
  ステータスバーに「上下キーで範囲、左右キー(Shiftでセル中央)で列位置を調整しEnterで確定、
  Escで取消」の操作案内が表示（`t048-04-retry.png`）。キーボードショートカット
  （Shift+F9）でも同様に起動することを確認。
- **配線分断（はさみ形）**：クリックで即座にセル位置へ配線分断を配置（1クリック完結型ツール）。
  同一セルへ再度クリックすると「この位置には既に配線分断があります」という重複検出メッセージが
  表示され、**1回目のクリックで正常に配置が成立していたことをデータ的に裏付ける**
  （`t048-06-f10retry.png`）。

なお、いずれのボタンも直前にEscでツールを解除した直後の1回目クリックが不発になり、
2回目のクリックで確実に反応する挙動を観測した（両ボタンで再現）。UI Automation合成クリック
特有のタイミング事象の可能性が高く、T-048の変更（Path Dataのみ）に起因するものではないと
判断し、深追いしていない。実操作（物理マウス）でも同型の遅延が起きるかは未検証。

## 総合判定

**T-048の4観点すべてOK。回帰なし。** 隠密静的レビュー（Trigger類無傷・便乗変更なし）と
実機結果が整合。上記のEsc直後1回目クリック不発の事象は範囲外の気づきとして記録するのみ
（実害はスクリーンショットで確認した限りなし、2回目で確実に成功）。
