# T-049 実機確認：デバイス名編集中の未確定編集を確定してから保存

検証日: 2026-07-10（忍者 key=1783637541231）
対象コミット: `bfa37cc`（隠密静的レビュー`docs/archive/ecad2-t049-review-onmitsu.md`＝要修正なし）

## 検証方法

`ecad2-ui-automation`スキル使用。`dotnet build`0警告0エラー確認後起動。判定は見た目でなく
**保存ファイル(.gcad、JSON形式)の内容**で行った。

準備：新規シート作成→a接点配置→初回Save As（`sample\t049-ninja-test.gcad`、gitignore対象で
コミット汚染なし）でファイルパス確立。以降のCtrl+Sはダイアログを介さず直接上書き。

各観点とも、プロパティパネルの「デバイス名」欄（`DeviceNameBox`）へ**クリックしたまま
フォーカスを保持した状態**（Tab移動・他要素クリックをしない）で新しい名前を入力し、
その未確定状態のままショートカットを送出した。

## 観点別判定

### 観点1: フォーカス保持のままCtrl+S → 保存ファイルに反映（判定：**OK**）

デバイス名欄に「Y5」を入力（フォーカス保持、機器表は未更新のまま＝未確定を確認）→
Ctrl+S送出 → 機器表が即座に「Y5 / Relay」へ更新（コミット確認）→ **保存ファイルを読取り、
`"deviceName": "Y5"`・`"devices":{"byName":{"Y5":{...}}}`が正しく書き込まれていることを
データで確認**。

### 観点2: フォーカス保持のままCtrl+N → 破棄確認ダイアログ（判定：**OK**）

デバイス名欄を「Y6」に再編集（フォーカス保持、未確定）→ Ctrl+N送出 →
**「現在のドキュメントには保存されていない変更があります。保存しますか？」の確認ダイアログが
正しく表示**（`t049-07-ctrln-dialog.png`）。ダイアログ背後の機器表も「Y6 / Relay」へ更新済みで、
未確定編集がIsDirty判定に正しく取り込まれたことを確認。キャンセルで中断し文書を保持。

### 観点3: フォーカス保持のままウィンドウクローズ → 破棄確認ダイアログ（判定：**OK**）

デバイス名欄を「Y9」に再編集（フォーカス保持、未確定）→ タイトルバー×ボタンでクローズ試行 →
**同様の確認ダイアログが正しく表示**（`t049-10-closeattempt.png`、機器表「Y9 / Relay」反映済み）。
キャンセルでクローズを中断、プロセス生存を確認。

### 観点4: 回帰確認（従来のフォーカス移動による確定経路）（判定：**OK**）

デバイス名欄を「Y8」に編集後、キャンバスの別セル（空）をクリックしてフォーカスを外す

`"deviceName": "Y8"`反映を確認。既存の確定経路に回帰なし。

### 補足: Ctrl+O（観点2と同一ゲートウェイだが個別確認）

デバイス名欄を「Y10」に再編集（フォーカス保持）→ Ctrl+O送出 → 同様の確認ダイアログが正しく
表示（`t049-11-ctrlo-dialog.png`、機器表「Y10 / Relay」反映済み）。隠密レビュー観点2の

という所見と実機結果が一致。

## 範囲外の気づき（T-049の変更点とは無関係、報告のみ）

**プロパティパネルの「要素を選択してください」プレースホルダが、要素配置直後に
更新されないケースがある。**

検証中、あらかじめ選択済みのセルにF5で要素を配置した直後、プロパティパネルが
「要素を選択してください」のプレースホルダのまま変化せず、`DeviceNameBox`が表示されない
現象に遭遇した（配置前後で`SelectedCell`の値自体は変化しないため）。一度別セルへ選択を
移してから元のセルへ戻すと正常にプロパティパネル（種別・デバイス名欄）が表示された。

静的に確認した限り、`MainWindowViewModel.SelectedCell`のsetter（`MainWindowViewModel.cs:185-223`）
は`SetProperty`が値変化を検知した場合のみ`HasSelectedElement`等をOnPropertyChanged通知する
設計で、要素配置処理自体（`MainWindowViewModel.cs:1370`付近）も配置後にこれらのプロパティを
個別通知していない。このため「空セルを選択→そのセルへF5配置」という一般的な操作順序では、
配置直後にプロパティパネルが更新されない可能性がある（本検証はT-049の差分に起因するものではなく、
既存コードの挙動として静的に確認したのみ。実機での再現条件の網羅的な切り分けは行っていない）。

## 総合判定

**T-049の4観点＋補足1件、すべてOK。隠密静的レビュー（要修正なし）と実機結果が完全に一致。**
回帰も確認できず。上記の範囲外気づき1件を除き、追加の懸念なし。
