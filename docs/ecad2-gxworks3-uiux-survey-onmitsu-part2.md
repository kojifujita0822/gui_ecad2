# GX Works3 実機UI/UX調査（T-012・隠密、T-010の実機検証版）

> 2026-07-03 隠密調査。殿がGX Works3体験版を入手し、実機でのUI/UX追加調査(T-012)を指示。
> T-010（Web一次情報ベース）の推定を実機スクリーンショット3枚＋インストールフォルダの
> ファイル解析で検証した。ユーザー提供のスクリーンショットを分析。

## 調査方法

- ユーザーが体験版（sw1dnd-gxw3-j_126g_try.exe）をインストールし、GX Works3(GXW3.exe)を起動
- 実行環境の制約（対話デスクトップから分離されたセッション）によりスクリーンショット取得・UI自動操作は隠密側から直接実行できず、ユーザー提供のスクリーンショット3枚を分析
- 補助的にインストールフォルダ（C:\Program Files (x86)\MELSOFT\GPPW3）のファイル解析を実施

## T-010との差分・検証結果

### 1. 画面構成（T-010の推定は概ね正確）
- 起動直後: メニューバー→ツールバー→中央キャンバス＋左「ナビゲーション」パネル（プロジェクトツリー）＋左下「クイック検索」＋右「部品選択」パネル＋右下「構成詳細情報入力」パネル＋下部「進捗/出力」パネル、が確認できた。T-010の推定通り。
- ラダー編集時は右パネルが「部品選択」から**コンテキストに応じて「ガイダンスフロー」（操作手順ガイド）に切替**。パネルの中身が状況依存で変わる点はT-010未言及の新発見。
- ツールバーはラダー編集時に**2段構成**になる：1段目は汎用操作（新規/保存/元に戻す等）、2段目はラダー専用コマンド。

### 2. 配色（T-010「白/グレー基調」を精緻化）
- **起動直後・プロジェクト未作成の空状態**: キャンバス及び各パネル内部の地色が**濃紺（ネイビーブルー）**。T-010の「白/グレー基調」は不正確だった可能性。
- **実際にラダープログラムを開いた作業領域**: 背景は**白＋黒グリッド線**。T-010の「白/グレー基調」はこちらに一致。
- → 結論: 「待機/空状態は濃紺、実作業領域（ラダーエディタ内部）は白」という**状態依存の配色**が実態。T-010は作業領域の配色を正しく捉えていたが、空状態の濃紺の存在は見落としていた。
- 選択中セルの枠線・ハイライトは**青**。未入力デバイス名は「?」を**黒字**で表示。

### 3. アイコン様式（T-010「不明」だった点を一部補強）
- ツールバーアイコンは小型（16-20px相当）、多色使い、フラット寄り。T-010の推定と概ね一致。
- **新発見**: ラダー編集ツールバーの各アイコン下に**ファンクションキー番号（F5, F6, F7...Shift+F5等）が直接明記**されている。マウスのツールバークリックとキーボードショートカットが視覚的に一体案内される設計。GuiEcadには無かった発想で、ecad2のキーボードファースト設計に直接応用できる有力パターン。

### 4. 操作フロー・インタラクション（T-010未調査の新規発見）
- 接点等のラダー記号をキャンバスに配置すると、**セル直下に浮動インライン入力ボックス**（デバイス名テキスト欄＋アイコン2つ）が即座に表示される。GuiEcadの課題A5（「Enterで配置後、機器名を直接入力できない」）に対する実例回答であり、R1・R6（フォーカス一元性・確定/取消の到達性）の具体的な実装パターンとして参考になる。
- 未保存の変更があるタブ・ツリー項目は名称が**赤字**になり、タブ末尾に`*`が付く（例: 「プログラム本体」ツリー項目、タブ「ProgPou [PRG] [LD] 1ステップ *」）。変更状態の視覚的一貫性という観点で参考になる。
- ステータスバーは機種名(FX5U)・局番・ステップ数・上書き/挿入モード・CAP/NUMを一列表示。情報密度が高い。

### 5. ファイル解析（インストールフォルダ）で判明した点・空振りだった点
- インストールフォルダには.csv(2630)/.dll(1237)/.h(1133)/.jpg(904)/.bmp(801)/.msg(788)等、多数の外部リソースファイルが存在。UIリソースの多くは実行ファイル外に配置されている。
- 「ColorFontDialog.db」というSQLite DBを発見し配色定義を期待したが、中身は`limit_target`テーブル（PC種別ごとの機能制限マスク情報）で、UI配色とは無関係だった（空振り）。
- 外部bmpアイコン群の大半はオンライン監視・システムモニタ等の専用ダイアログ用で、メインツールバーのアイコン画像は実行ファイル（GXW3.exe等）内蔵リソースの可能性が高く、抽出には専用ツール（Resource Hacker等）が必要。今回は費用対効果を考慮し深追いしなかった。

## 総合結論（T-008/T-010裁定との整合）

殿裁定済みの「区画分けは維持、中身は視覚シンプル化＋キーボード主体・ボタン補助」の方向性と、実機観察は矛盾しない。特に**ツールバーアイコンへのファンクションキー番号併記**と**配置直後の浮動インライン入力ボックス**の2点は、ecad2のUI設計に直接転用できる具体的パターンとして推奨する。配色は「空状態=濃紺、作業領域=白」という状態依存の使い分けを踏襲するか、ecad2独自にシンプル化するかは今後のUI/UXブリーフでの検討事項とする。

## 不明点（今回も未解明）
- ツールバーの正確なボタン数（画像から数えられる範囲に留まる）
- ラダー要素（接点・コイル）のオンラインモニタ中の導通色（今回のスクリーンショットはオフライン編集画面のみ）
- メインツールバーアイコンの線画/塗りつぶしの技術的実装詳細
