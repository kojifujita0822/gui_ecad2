# GX Works3 UI/UXデザイン調査（T-010・隠密）

> 2026-07-03 隠密調査。殿より「ecad2のUI視覚デザインをGX Works3（三菱電機PLCエンジニアリングソフト）
> 踏襲イメージで具体化したい」との追加指示を受け実施。ローカルにGX Works3本体は無いため、
> Web上の一次情報（三菱電機公式マニュアル・製品サイト・公式カタログ）を情報源とした。

## 結論

GX Works3のUIは**クラシック・実務優先型**（Windows 2000年代的なメニューバー＋複数専用ツールバー構成）。
リボンUIではなく、機能を漏れなくメニューツリーに収める「詰め込み型」設計。配色は白/グレー基調で
控えめだが、モニタ実行時の状態表示（導通=青／非導通=黒など）は明確に色分けされる。

## 根拠

### 1. 画面構成
メニューバー→ツールバー→作業領域→ドッキングウィンドウ→ステータスバーの伝統構成。左上に
ナビゲーション（プロジェクト）ツリー、右側に要素選択ウィンドウ、下部中央に出力ウィンドウ、
中央にラダーエディタ。全ウィンドウはドッキング可能。
出典: [GX Works3 Operating Manual SH-081215ENG p.50-71](https://dl.mitsubishielectric.com/dl/fa/document/manual/plc/sh081215eng/sh081215engan.pdf)

### 2. アイコンの様式
小さめ（16〜24px相当）、限定的な多色使い。線画/塗りつぶしの別は断定できず。古いカタログでは
若干グラデーション調も見られるが、新しい画面はフラット寄り。
出典: 三菱公式カタログ、「教えて！GX3号」記事画像

### 3. 配色
基調は白/グレー。モニタ中の導通表示は青、非導通は黒（マニュアル本文で明記）。差分照合は
ピンク/青/緑で色分け。エディタ配色はユーザーカスタマイズ可（黒背景化も可能）。ラダー要素
（接点・コイル）自体の既定色表は未確認＝不明。
出典: 同マニュアル14.2章・2.7章、[GX Works3 デバッグ機能ページ](https://www.mitsubishielectric.co.jp/fa/products/cnt/plceng/smerit/gx_works3/debug.html)

### 4. ボタン/ツールバーの密度
単一ツールバーでなく用途別（Standard/Program/Monitor Status等）に複数用意され、切替使用。
メニューは多階層で網羅的。右クリックはプロジェクト管理系では補助的に用意されるが、ラダー編集の
中核操作（接点・命令挿入）ではツールバー/ダイアログ/ドラッグ&ドロップが優先で、右クリックへの
依存度は低い。

**不明点**: ツールバーの正確なボタン数、アイコンの線画/塗りつぶし別、ラダー要素個別の既定色、
ステータスバーの詳細表示項目。

## 推奨案

ecad2には、リボンUIではなく「メニューバー＋複数専用ツールバー＋ドッキング可能パネル」の
クラシック構成を踏襲し、左にプロジェクトツリー、下部に出力/エラーパネルを配置するのが妥当。
配色は白/グレー基調を基本としつつ、モニタ時の導通=青／非導通=黒という三菱の慣習的配色は
現場エンジニアの馴染みが強いため踏襲を推奨。頻用操作（接点・コイル挿入）はツールバー/
キーボード優先でアクセスさせ、右クリックは補助的機能に留めるのがGX Works3の設計思想と整合的。

なお、殿裁定済みの「区画分けは維持、中身は視覚シンプル化＋キーボード主体・ボタン補助」の方向性
（T-008）とも矛盾なく両立する。GX Works3のクラシック構成（区画分け）を土台にしつつ、
アイコン密度・配色は現代的にシンプル化する折衷が妥当と考えられる。
