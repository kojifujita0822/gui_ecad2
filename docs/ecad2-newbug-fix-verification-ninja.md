## 観点(A) 新規バグの再現確認(配置操作中の文字視認性) → **OK、3回連続再現なし**

シート追加→「配置ツール」タブ→a接点配置ツール選択→キャンバスクリック(インラインダイアログ
表示)→シート項目色確認→Escキャンセル、を3回反復。

- `SheetNavList.IsEnabled`は3回とも`False`(既存仕様どおり、ダイアログ表示中は無効化される)
- 選択中シート項目の画素採取(支配色)は**3回とも完全一致**——背景`#0078D7`(選択色)、
  **文字`#FFFFFF`(白、修正前は`#6D6D6D`グレーだった)**。低コントラスト問題は解消。

## 観点(B) T-089本来の観点の最終確認(App.xaml累積修正) → **OK**

ライト/ダーク両モードで、選択色・フォーカス無し選択・MouseOverフィードバックを画素採取で確認。

| 項目 | ライトモード | ダークモード |
|---|---|---|
| 選択色(背景/文字) | `#0078D7`/`#FFFFFF` | `#0E639C`/`#FFFFFF` |
| 改名直後(フォーカス無し選択) | `#0078D7`/`#FFFFFF` | (下記補足参照) |
| MouseOver背景(実測/理論値) | `#E5F3FB`/`#E5F3FA` | `#2D3B45`/`#2C3B45` |

いずれも期待値と一致(理論値との誤差は1程度、丸め誤差の範囲)。

**補足(範囲外・cc0402bとは無関係の既知現象)**: ダークモードでの改名操作時、対象がリスト
1番目の項目だと改名確定直後に`IsSelected=False`となる現象を今回も観測(T-089検証時に発見済みの
既知の気づき、`docs/ecad2-t089-followup-mouseover-selection-verification-ninja.md`「範囲外の
気づきA」と同一)。`SelectionItemPattern.Select()`で明示的に再選択したところ`IsSelected=True`
に戻り、選択色(`#0E639C`/`#FFFFFF`)は正しく表示された。本件は`cc0402b`の修正対象
(IsEnabled時のグレー文字化)とは無関係の別問題であり、既に把握済みにつき新規報告はしない。
