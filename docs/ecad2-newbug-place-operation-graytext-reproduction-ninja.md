# 新規バグ再現性確認: 配置操作時のシート・出力パネル文字色変化

忍者作成、2026-07-20。家老委譲(殿発見の新規バグ、コミット`a1f4646`回帰疑い)の再現性確認結果。

## 症状

要素配置ツール(a接点等)を選択しキャンバスをクリックしてインライン配置ダイアログを表示している間、
シートパネルの選択項目・各タブヘッダー(基本機能/配置ツール/シート/出力)の文字が見えにくい
グレー系の色に変化する。

## 再現性: **100%(3回連続再現)**

シート追加→a接点配置操作(キャンバスクリックでインラインダイアログ表示)→キャンセル、を3回
繰り返したところ、**毎回完全に同一の結果**が得られた。

- `SheetNavList`の`IsEnabled`: ダイアログ表示中は3回とも`False`(ダイアログキャンセル後は
  3回とも`True`に復帰)
- シート選択項目の画素採取(支配色): 3回とも完全一致——背景`#0078D7`(選択色、正常)、
  文字`#6D6D6D`(3回とも同一のグレー、期待値の白`#FFFFFF`ではない)
- 「出力」タブラベル部分にも同系統のグレー(`#6D6D6D`)を検出

## 原因の手がかり(参考情報、原因特定・修正は侍・隠密に委ねる)

- UIA実測で、配置ダイアログ表示中は`SheetNavList`・全`TabItem`(基本機能/配置ツール/シート/出力)
  が`IsEnabled=False`になることを確認。メインウィンドウ自体(`Get-Ecad2Root`)は`IsEnabled=True`
  のまま。
- `MainWindow.xaml`827-842行目のコメントによれば、これは**T-033増分2で導入された既存の意図的
  仕様**——配置操作中は`MainContentArea`(Menu/メイン作業域/OutputPanelAreaを束ねたラッパーGrid)
  全体を`IsMainContentEnabled`バインドで無効化する設計であり、「配置バー・ステータスバーは
  グレーアウト対象外」という殿裁定の記載もある。**無効化自体は新規バグではなく既存の意図的挙動**
  と見られる。
- 一方、`App.xaml`のListBoxItemスタイル(T-089往復3周目、未コミット差分)には
  `<Trigger Property="IsEnabled" Value="False"><Setter TargetName="Bd"
  Property="TextElement.Foreground" Value="{DynamicResource {x:Static
  SystemColors.GrayTextBrushKey}}"/></Trigger>`という新設トリガーがある。このトリガーは
  `IsSelected`トリガー(Background=選択色のまま)より後に定義されており、**「背景は選択色の
  ままなのに文字だけグレーに変わる」という低コントラストな組み合わせ**を生んでいると推察される
  (WPFトリガー適用順序で、同一プロパティでない場合は両トリガーの効果が併存するため)。
- 推察が正しければ、本症状はT-089修正(App.xaml、IsEnabled=Falseトリガー追加)と、既存の
  T-033増分2グレーアウト機構(MainWindow.xaml、IsMainContentEnabled)が**組み合わさって初めて
  顕在化する**副作用であり、単独のコミット`a1f4646`(MainWindow.xaml/csのみ変更、App.xaml変更を
  含まない)だけが原因ではない可能性がある。

## 検証環境

- ビルド: 現在のworking tree(App.xaml未コミット差分+コミット`a1f4646`込み)、0警告0エラー
- 再現手順: シート追加→「配置ツール」タブ→a接点配置ツール選択→キャンバスクリック(インライン
  ダイアログ表示)→(この状態でシートパネル・タブラベルの文字色を確認)→Escでキャンセル、を反復
