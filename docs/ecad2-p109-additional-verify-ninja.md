# P-109 追加実機検証（忍者、2026-07-22）

## 対象
`docs/proposed.md` P-109（シート改名で選択色消失）の追加検証。隠密の原因調査
（`docs/ecad2-p104-p109-cause-investigation-onmitsu.md`）を受け、家老采配。
(1) 物理クリック（UIA合成操作との差異切り分け）(2) 2番目以降のシートで本当に発生しないか
複数回試行での再確認、の2点。

## 操作手法
`Invoke-Ecad2ScreenClick`（合成マウスイベント、`SetCursorPos`+`mouse_event`）と
`SelectionItemPattern.Select()`（UIA）の両方を使用。殿ご自身の実操作による確認は未実施
（本検証で得られた新事実により、その必要性は薄れたと判断——後述）。

## 結果1：UIA固有ではないと確認（合成マウスクリックでも再現）

1番目シートを`Invoke-Ecad2ScreenClick`（座標クリック、ListBoxItem実座標）で選択→名前変更→OK確定
したところ、UIA `SelectionItemPattern.Select()`と同様に選択色が消失した。**入力経路（UIA Select
呼び出しか、合成マウスクリックか）によらず同じ症状が発生**——隠密の判断「UIA固有のバグではなく
アプリ側の実装構造に起因する可能性が高い」を実証的に裏付けた。

## 結果2：時間経過での自然回復なし

1番目シート改名確定直後・500ms後・1500ms後の3時点で`IsSelected`を追跡したが、いずれもFalseの
まま。1.5秒待っても回復しなかった。隠密が懸念した「`ContextIdle`遅延実行がUIA読み取りタイミングに
間に合っていないだけ」という説明は、この結果からは支持されない（選択状態は一時的な読み取りズレ
ではなく、実際に失われたまま復帰しない）。

## 結果3：【重要な訂正】「1番目限定」ではなく「リスト末尾以外で発生」

複数のシート枚数構成・複数の対象位置で改名操作を繰り返したところ、当初の原記録・忍者の先の
再現確認（`docs/ecad2-p104-p109-reproduction-check-ninja.md`）で見えていた「1番目限定」という
条件は不正確で、正しくは**「改名対象がリストの末尾（最後の項目）でない場合に選択色が消失する」**
という条件であることが判明した。

| 試行 | シート構成 | 対象位置 | 結果 |
|---|---|---|---|
| UIA Select・1周目 | 2枚 | 1番目（非末尾） | 消失 |
| UIA Select・1周目 | 2枚 | 2番目（**末尾**） | **正常維持** |
| 合成クリック | 3枚 | 1番目（非末尾） | 消失 |
| 合成クリック | 3枚 | 2番目（非末尾） | 消失 |
| 合成クリック | 3枚 | 3番目（**末尾**） | **正常維持** |
| UIA Select・2周目 | 3枚 | 1番目（非末尾） | 消失 |
| UIA Select・2周目 | 3枚 | 2番目（非末尾） | 消失 |
| UIA Select・2周目 | 3枚 | 3番目（**末尾**） | **正常維持** |
| UIA Select | 4枚 | 3番目（**今回は非末尾**） | **消失** |

**決定的な証拠**：3枚構成で「末尾」だった3番目のシートを、4枚目追加後に同じ3番目（今回は非末尾）
として再度改名したところ、今度は選択色が消失した。**同一シートが、リスト内の位置（末尾か否か）
によって挙動を変える**ことを直接確認した。入力経路（UIA/合成クリック）を問わず一貫している。

### 所見
この「末尾/非末尾」という条件は、隠密が特定した構造（`Sheets.RemoveAt(index); Sheets.Insert(index,
sheet);`）と整合的に見える——末尾要素の削除・再挿入は後続要素のインデックスに影響しないが、
非末尾要素の場合は後続の全要素のコンテナ管理に波及する、という違いが機序に関わっている可能性が
考えられる。ただしこれは忍者の推測であり、断定はしない（機序の特定は隠密・侍の領分）。

## 総括

| 観点 | 結果 |
|---|---|
| UIA固有か | 否——合成マウスクリックでも同一症状、実装構造起因と裏付け |
| タイミング問題か | 否——1.5秒待っても自然回復なし |
| 「1番目限定」の正確性 | **訂正要**——正しくは「末尾以外で発生」、同一シートで位置依存を実証 |

物理クリック（合成マウスイベント）で実装構造起因と裏付けが取れたため、殿ご自身の実操作での
追加裏取りは今回は不要と判断する（家老の判断を仰ぐ）。
