# T-015 スコープ再定義：殿への選択肢提示（隠密起草）

T-015は元々「左パレットの図形ビジュアルプレビュー」だったが、T-026で図形選択が左パレットから
ツールバー（固定6種）＋右パネル「部品選択」リスト（固定6種＋自作パーツ全Entries）へ移行済みのため、
プレビュー対象自体が消滅している。着手前に対象を再定義する必要がある（UI/UX分岐、実装着手はしない）。

## 現状のUI（T-026後）

- **ツールバー固定6ボタン**（a接点/b接点/コイル/端子台/OR a接点/OR b接点、`MainWindow.xaml` 145-201行）:
  ボタン自体にXAML `Path` で描いた簡易記号アイコンが既にある（見た目の判別は一定程度可能）。
- **右パネル「部品選択」リスト**（`PartSelectionList`、`MainWindow.xaml` 320-336行、`PartPalette.Entries`）:
  「自作パーツ」ボタン押下時、`Tool.Mode==PlaceElement`の間表示される。固定6種＋自作パーツ
  （`PartFolderStore.Enumerate()`で読み込み）を**同一リストに列挙**するが、表示は
  `Category`（テキスト）＋`Definition.Name`（テキスト）のみで、**図形そのもののビジュアルは一切ない**。
  自作パーツは名前だけでは形状を判別できない。

## 技術的土台（既調査、`docs/ecad2-preimplementation-survey-onmitsu.md`）

`DiagramRenderer.DrawPreview(IRenderer r, ElementInstance e, Color color)`（1要素だけを指定色で
描く既存メソッド、配置ゴースト表示用）を流用し、`DrawingVisual`→`RenderTargetBitmap`化する薄い
ラッパーを新規に書けばサムネイル生成が可能（SVG経由よりコスト低、と結論済み）。

## 選択肢

### 案1（本命）: 右パネル「部品選択」リストの各項目にサムネイルを追加

- **内容**: `PartSelectionList`の`ItemTemplate`にサムネイル画像（`DrawPreview`→`RenderTargetBitmap`）
  をテキストの左に追加。固定種別・自作パーツとも同じリストなので両方に効く。
- **利点**: T-015原案の目的（見た目で図形を判別）に最も直接的に応える。特に自作パーツは名前だけ
  では区別しづらく効果が大きい。既存調査どおり実装コストが低い。
- **コスト/規模**: 中。新規実装は「単一要素をオフスクリーン描画してビットマップ化する」ラッパー層
  のみ（描画ロジック自体は`IRenderer`/`DrawPreview`を流用）。生成タイミング（起動時一括／リスト
  表示時に遅延生成）の設計判断が要る。

### 案2: 案1に加え、選択中1件を拡大表示する専用プレビューエリアを新設

- **内容**: 部品選択リストの上または横に固定プレビューペインを追加し、リスト選択変更に連動して
  拡大表示する。
- **利点**: 小さいサムネイルでは判別しづらい複雑な自作パーツも拡大確認できる。GX Works3等でも
  よく見るパターン。
- **コスト/規模**: 案1より高い。新規UI要素の追加、選択変更イベントとの連動実装が増える分、右パネル
  のレイアウト（既存の上下2分割構成）にも影響しうる。

### 案3: ツールバー固定6ボタンのツールチップにも実描画プレビューを追加

- **内容**: 現状テキストのみのToolTipを、`DrawPreview`によるレンダリング画像に差し替える。
- **利点**: 配置前に実際の見た目を確認できる一貫性。
- **コスト/規模**: 低〜中。ただし固定6種は既にXAML `Path`記号アイコンがボタン自体にあり判別は
  一定程度可能なため、**追加による効果は他案より限定的**（対効果が薄い参考案）。

## 隠密所見

案1が本命（対効果が高くコストも中程度）。案2は視認性は最良だがレイアウト変更を伴いコストが増す
ため、案1導入後に手応えを見て追加検討でもよい。案3は固定種別に対しては既にアイコンがあるため
優先度は低い。対象範囲（固定種別も含めるか自作パーツのみに絞るか）も殿の選択肢に含めるとよい。
