# T-021 増分(v)追加修正（コミット340f53d）静的レビュー

隠密によるレビュー。対象: `src/Ecad2.App/MainWindow.xaml.cs` MoveSelectedCell、
コミット340f53d（矢印追従スクロールで端到達時に母線側へ範囲を拡張、殿直接指示）。
`code-review`スキル（medium、8観点finder→1-vote verify）を併用。

## 手動観点の確認結果（Phase 0）

- **母線位置との整合性**: `DiagramRenderer.BusPad = Cell*0.5`（左母線=X(0)-BusPad、右母線=X(Columns)+BusPad）に対し、
  今回の拡張量は1.0Cell（BusPadの2倍）。母線を確実に内包する安全マージンとして機能し、正しさの問題はない。
- **単一列グリッド(Columns==1)の縁ケース**: 両条件(`viewColumn==0`と`viewColumn==Columns-1`)が同時成立し
  幅が3倍になるが、実際に計算すると左右母線とも均等マージン0.5Cellで対称に収まる（後述のverifyでも確認）。
- **既存の右母線余白送り対策(viewColumnクランプ)との整合性**: 新規判定はクランプ後の`viewColumn`を使っており、
  クランプの意図(右母線への過剰スクロール防止)と矛盾しない。
