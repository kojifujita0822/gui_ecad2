# T-051前提検証：Ctrl+Z（Undo）実機確認

検証日: 2026-07-10（忍者 key=1783637541231）

## 背景

家老（侍の調査＋独立grep）により「ecad2にUndo機構は未実装（UndoStack・Ctrl+Z配線とも不在、
メニュー項目は表示のみ未配線）」と判明。一方、過去の忍者観測2件がこれと矛盾する：

- **P-032**（T-041増分5実機確認）：「要素の配置・削除はUndo可能なことを他場面で確認済み」との対比記述
- **P-042**（T-045増分D最終実機検証）：「通常のキャンバス内ドラッグではCtrl+Z正常復元を別途確認」

いずれも本筋（シートUndo対象外・境界外ドラッグ）の**傍証として添えた一言**であり、Undo自体を
主目的とした厳密検証ではなかったと一次資料（`docs/proposed.md`）を確認して判明。今回はUndoの
挙動そのものを主目的として、データ側で白黒つける。

## 検証手順・環境

- `ecad2-ui-automation`スキル使用、`dotnet build`0警告0エラー確認後に起動
- UI Automation Invoke（ボタン直接呼び出し）は配置ツールボタンで例外発生（シート0枚時は
  CanExecute=false由来と判明、既知の罠と整合）→ **SendKeys（キーボードショートカット）に切替**
  （このアプリはSendKeysが正常に届く、スキル0節記載どおり）

## 観点1：要素配置後のCtrl+Z（OK/NG判定：**NG**）

1. シート追加（AddSheetButton）→ セル（行1/列0）選択 → F5（a接点配置）→ 配置バーでEnter確定
   → **キャンバス上に接点記号(｜｜)が描画されることをスクリーンショットで確認**
   （`undo-check-06.png`）
2. `Send-Ecad2Keys "^z"` を実行 → **要素は消えず、キャンバス上に残存**（`undo-check-08-afterctrlz.png`）
3. 疑わしい結果のため2度追加でCtrl+Zを送出（計4回）→ **変化なし、要素は残存し続ける**
   （`undo-check-09-after3xctrlz.png`）
4. キーボードショートカットではなく**「元に戻す (Ctrl+Z)」ツールバーボタンをUI Automation
   Invokeで直接押下** → **これも要素は消えず残存**（`undo-check-10-afterbutton.png`）

**結論：4通りの試行（キーボード×3回+ボタン直接押下×1回）すべてで、配置した要素はCtrl+Zで
消えなかった。Undoは機能していないとデータ（スクリーンショット上の要素の実在）で確認。**

## 観点2：要素移動（ドラッグ）後のCtrl+Z（判定：**検証不能・別の事実を確認**）

- マウス合成ドラッグ（`SetCursorPos`+`mouse_event`、多段階移動で2回試行、うち1回は低速・
  高精度で再試行）で、配置済み要素のセルを起点にドラッグを実施
- **いずれの試行でも要素本体は移動せず、「選択セル」のみがドラッグ終点のセルへ移動した**
  （`undo-check-11-afterdrag.png`／`undo-check-12-drag2.png`：接点記号は列0に残存、赤枠の
  選択セルのみ列5へ移動）
- このためマウスドラッグ経由での「要素移動」を再現できず、移動後のCtrl+Z検証（観点2本来の
  手順）には進めなかった
- 補足：P-042の原観測はFreeLine（自由線）のドラッグに関するもので、今回試したのはa接点
  （基本要素）。要素種別が異なるため、この検証不能はP-042とは直接対応しない

## 副次所見（データ的傍証、範囲外）

- **「元に戻す」「やり直し」両ボタンとも、シート0枚・操作履歴皆無の起動直後からIsEnabled=True**
  （観点1の操作前に確認済み）。Undoスタックが空の状態でボタンが有効なのは、CanExecuteが
  履歴と連動していないことを示す状態的傍証で、観点1の実測結果と整合する
- ステータスバーの「シートがありません。新規作成（Ctrl+N）から始めてください」というメッセージが、
  シート追加後も消えずに残り続ける事象を観測（`undo-check-03〜10`で継続）。実害は未確認・
  範囲外の気づきとして記録のみ（本検証の主目的ではないため深追いせず）

## 総合判定

**家老の「Undo機構は未実装」という所見と、忍者の実機確認（観点1）は完全に一致する。**
P-032・P-042の該当記述（要素配置Undo確認済み／通常ドラッグUndo正常復元確認）は、一次資料
（proposed.md）を確認する限り、本腰の検証ではなく別の主題調査中に添えた傍証的な一言であり、
今回の厳密な実機再検証（4試行）の結果と矛盾する。**忍者として、過去のP-032・P-042該当箇所の
記述は見かけ上の誤認であったと訂正する。**

観点2（移動後Undo）はドラッグそのものが再現できず検証未了。ただし観点1で「Undo機構自体が
機能していない」ことが確定した以上、移動操作のUndoも同様に機能しない可能性が高いと推測される
（未検証・断定はしない）。
