# T-056 グリッド線表示切替の事前案起草

調査者: 隠密2　最終更新: 2026-07-11

殿裁定2026-07-10によりApproved（gated、T-052〜054の後）。着手時期はまだだが、家老采配により
事前の見せ方・実装コスト調査を先行して行う。コード変更は伴わない案起草のみ。

---

## 現状（調査結果）

- `src/Ecad2.App/Views/LadderCanvas.cs:29-32`：`_renderer`が`private readonly`フィールドで
  コンストラクタ時に`new DiagramRenderer(options: new RenderOptions { ShowGrid = true })`と
  **固定生成**されている（T-030「グリッド線表示の有効化」殿裁定2026-07-03、実機テストの行合わせ
  補助として常時表示化した経緯）。**現状は実行時に値を切り替えられない構造**——`ShowGrid`を
  動的に切り替えるには、このフィールドの`readonly`解除・値変更時の再描画トリガー（`InvalidateVisual`
  等）の追加が必要。
- **メニュー項目は既に存在する**：`src/Ecad2.App/MainWindow.xaml:129-131`の「表示(_V)」メニューに
  `<MenuItem Header="グリッド表示(_G)"/>`が既にあるが、`Click`/`Command`/`IsCheckable`のいずれも
  未設定（他の未結線項目=PDF出力等と同じ欠落パターン）。**器は用意済みで結線のみが不足**。
- **トグル型UIの既存先例は皆無**：`ToggleButton`/`IsCheckable`/`IsChecked`を全ソースでGrepしたが
  該当なし。チェック可能なメニュー項目・押下状態を保持するボタンはecad2にまだ一例もない
  （新規UIパターンの導入になる）。
- **アプリケーション全体のユーザー設定永続化機構が存在しない**：`Properties.Settings`・
  `ApplicationSettings`・`UserSettings`・`appsettings`のいずれも全ソースでGrepしたが該当なし。
  永続化はドキュメント（`.gcad`）側の`DocumentSettings`（`EnableBorder`等）にのみ存在し、
  アプリ全体のビュー設定（ウィンドウ位置・パネル幅・グリッド表示等）を保存する仕組みは
  T-059のGridSplitterパネル幅も含め**現状すべて非永続**（再起動でリセット）。
- **既存ショートカット一覧**（`MainWindow.xaml`全体を確認、衝突確認の母集団）：
  `Ctrl+N/O/S/P`（PDF出力は予約済だが未結線）・`Ctrl+Z/Y`・`Ctrl+X/C/V`・`F5/F6/Shift+F5/Shift+F6`
  （ラダー要素配置、恐らくF7/F8も同系統）・`Esc`（選択ツール）・`Ctrl+Shift+Up/Down`（行追加/削除）。
  **`Ctrl+G`は現行未使用**——GX Works3系CADツールでグリッド表示切替に慣例的に使われるキーであり、
  衝突なく採用できる候補。

## 案（配置候補）

**案A: 「表示(_V)」メニューの既存項目に結線（実装コスト: 小、推奨）**

`MainWindow.xaml:130`の`<MenuItem Header="グリッド表示(_G)"/>`に`IsCheckable="True"`と
`IsChecked="{Binding IsGridVisible, Mode=TwoWay}"`（または`Command`+`ToggleGridCommand`）を追加、
`InputGestureText="Ctrl+G"`も付記。VM側に`IsGridVisible`プロパティを新設し、`LadderCanvas`側の
`ShowGrid`と連携させる（`readonly`解除＋再描画トリガーの追加要）。

- 長所: メニュー項目が既に用意されており、結線のみで完結。新規UI要素の追加が不要。
- 短所: メニューを開かないと現在の表示状態が視覚的に分からない（チェックマーク以外の常時表示の
  手がかりがない）。

**案B: ツールバーへトグルボタン新設（実装コスト: 中）**

ツールバー1段目（`ToolBarArea`Band=0）にトグル型ボタン（`ToggleButton`スタイル新規作成）を追加。

- 長所: 常時見える位置にあり状態が一目でわかる。他の頻用操作（新規/開く/保存等）と並ぶ動線。
- 短所: **トグル型ボタンの既存スタイル資産が皆無**のため、新規に`ToggleButtonStyle`を起こす必要が
  あり実装コストが増す（押下状態の見た目=枠色や背景変化等を新規デザインする必要）。

**案C: 案A＋案Bの併用（実装コスト: 中〜大）**

メニューとツールバーの両方に置き、状態を`IsGridVisible`一つで共有する。

- 長所: メニューからもツールバーからも操作可能、視認性と操作性を両立。
- 短所: 実装コストは案Bの新規トグルスタイル分がそのままかかる。

### 隠密所感

**案Aが最小コストで最も有力**——メニュー項目が既に用意されている以上、まず結線を通してからツールバー
ボタンの要否を判断する順が合理的と考える。ただしメニューのみだと即応性に欠けるため、頻用が想定される
なら案Cへの拡張余地を残す設計（`IsGridVisible`をVMに1つ持つ形）にしておくのが無難。決定は殿に委ねる。

## 既定値論点

現行はハードコードで`ShowGrid=true`（T-030、殿裁定＝実機テストの行合わせ補助として常時表示化した
経緯）。切替機能導入後の既定値は、
1. **現状踏襲でtrue**（グリッド常時表示のまま、切替は必要な時だけ非表示にする用途）
2. **false**（GuiEcadや一般CADのデフォルトに寄せる、必要な時だけ表示）
のどちらにするかは、T-030の経緯（実機テストのための恒常表示）が現行運用でまだ必要かどうかに
依存する。着手時に殿確認が必要。

## ショートカット候補（衝突確認込み）

- **`Ctrl+G`**：現行未使用、衝突なし。GX Works3系CADツールでのグリッド切替慣例に近い
  （**推測**——ecad2側の他ショートカットとの衝突有無は確認済みだが、GX Works3実機での
  実際の割当キーそのものは本調査では未確認）。
- 代替候補として`Ctrl+;`（一部CADの慣例）もあるが、日本語配列キーボードでの押しやすさの観点で
  `Ctrl+G`に劣ると考える（隠密所感、実機確認は忍者マター）。

## 永続化要否

- **現状、永続化しない場合は追加実装コストゼロ**（アプリ起動時は常に既定値、他のビュー設定
  =パネル幅等と同様の非永続方針を踏襲するだけで済む）。
- **永続化する場合**、アプリ全体のユーザー設定を保存する仕組みがecad2にまだ一つも存在しないため、
  グリッド表示のためだけに新規の設定ファイル読み書き機構（JSON等）をゼロから設計・実装する必要が
  あり、実装コストは中〜大に跳ね上がる。T-059のパネル幅等、他のビュー設定と合わせて永続化基盤を
  まとめて設計するかどうかも論点になりうる（本調査の範囲外、気づきとして申し送り）。
- 永続化の要否・対象範囲（グリッド表示単体か、ビュー設定全般の基盤を作るか）は着手時に殿確認が必要。
