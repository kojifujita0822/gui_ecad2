# T-075 主要機能仕様書整備 領域分割案（隠密）

殿裁定（本日、忍者上申を受けたもの）に基づく`docs/ecad2-spec-{領域}.md`体系の着手前分割案。
**本書は分割案の提示のみ。各領域の仕様書本体はこの後、家老・殿の確認を経て着手する。**

対象調査：`MainWindow.xaml`のメニュー/ツールバー構成全体、レイアウト区画、`Ecad2.Core`構成、
`docs/todo.md`のタスク傾向をExploreエージェントへ委譲し俯瞰した（詳細はAgent報告に基づく）。

## 領域分割案

### 第1弾（最優先——殿指定の難航3件を含む）

1. **シート/ドキュメント管理** — 新規/開く/保存、シートナビゲーション、**シート初期状態
   （0件始まりの仕様）**、行数拡張、母線名、シート設定ダイアログ
2. **結線操作** — 自由線(横/縦)・縦分岐線・接続点・配線分断、**シート種別（主回路/制御回路）
   依存の結線ボタン有効/無効条件**
3. **Undo/Redo** — **対象範囲の明文化**（現状MVP＝シート構成のみ復元という限定仕様、忍者上申の
   直接の火種）

1と2は「シート種別による挙動差」という共通軸を持つため隣接着手が効率的、3は独立度が高いが
緊急性が並ぶため同弾に含める。

### 第2弾（メニュー構成全体——DoD4「将来のヘルプ画面流用」の基盤）

4. **メニュー・ツールバー全体構成** — 項目一覧、ショートカットキー体系、有効/無効条件、
   既知の不整合（例：F5がa接点配置とテストモードで表記重複＝T-061既知事項）を含む

以降の全領域仕様書から参照される「入口」に当たるため、第1弾（難航3件）の直後に早期整備する
価値がある。

### 第3弾（基本機能、使用頻度高）

5. **配置操作** — 要素配置(F5〜F8)、配置バー、部品選択との連動

### 第4弾（その他機能領域、優先度は着手時に再検討）

6. 機器表・BOM　7. 部品選択・自作パーツ管理　8. PDF出力（未結線だがCore層仕様は確定済み）
9. テストモード（同、未結線だがCore層仕様は確定済み）　10. 設計チェック(DRC)・出力パネル
11. キャンバス表示（ズーム・グリッド表示・配色）　12. ステータスバー・モード可視化

## 着手順序（提案）

**1. シート/ドキュメント管理 → 2. 結線操作 → 3. Undo/Redo → 4. メニュー・ツールバー全体構成
→ 5. 配置操作 → （以降、家老の采配で第4弾から順次）**

## 各領域仕様書の作成方針（DoD2・DoD4対応、着手時に適用）

- 実装コード（コメント含む）・`docs/todo.md`/`docs-notes/todo-archive.md`の殿裁定記録・
  `docs-notes/`配下の忍者実機検証記録を突き合わせ、「仕様として確定している挙動」を明文化する。
- 事実（コード上の実装）と裁定根拠（殿裁定の経緯）を区別して記載し、出典（ファイル:行番号／
  todo.mdの該当タスクID）を明記する。
- 将来のヘルプ画面流用・隠密自身の静的レビュー共通基準としても使える形式を意識し、機能名・
  条件・挙動を表形式等で構造化する（散文的な説明のみにしない）。

## 殿・家老への確認事項

1. 上記領域分割・着手順序でよいか。
2. 第1弾3件から着手してよいか（規模が大きいため、1領域ごとに進捗報告する運びとする）。
