# T-077増分1 実機検証（忍者）

**対象コミット**: `6063714`（「使い方」ウィンドウPoC：非モーダル・F1・FlowDocument変換）
**検証日**: 2026-07-21
**手法**: `ecad2-ui-automation`スキル。使い方ウィンドウはメインウィンドウとは別ウィンドウのため、`Ecad2Native`経由のPrintWindow撮影＋画素採取（背景色・文字色は色出現頻度集計方式）。

## DoD別結果

### (1) F1キーで「使い方」ウィンドウが開くこと — OK

`Send-Ecad2Keys "{F1}"`実行後、`EnumWindows`で新規ウィンドウ「使い方」の出現を確認（`ecad2-t077z1-f1-window.png`）。

### (2) 「ヘルプ」メニュー「使い方(_G)」項目クリックでも開くこと — OK

いったんウィンドウを閉じた（`WindowPattern.Close()`）のち、「ヘルプ」メニューを展開し「使い方(G)」項目をInvoke、正しく開くことを確認。

### (3) 非モーダルであることの確認 — OK

使い方ウィンドウを開いたまま、メインウィンドウ側で以下を実行し、いずれも正常に機能することを確認：
- シート追加ボタン→ダイアログ表示→OK確定
- a接点配置ツール選択
- キャンバスクリック→インライン配置ダイアログ→デバイス名入力（M1）→OK確定→要素配置完了、機器表へ反映

途中、使い方ウィンドウのウィンドウ座標がキャンバス領域と重なっていたため、物理クリックが背後のキャンバスへ届かず一瞬「反応しない」ように見えた場面があったが、`GetWindowRect`で座標を確認しウィンドウを移動させたところ問題なく操作できた。これは非モーダル実装の正しい挙動（`Owner=this`のみでZ-order制御を行わない設計、隠密レビュー(c)参照）であり、バグではない。

### (4) 多重起動防止（既存ウィンドウの前面化） — OK

使い方ウィンドウを開いたまま、F1キー再送信・「ヘルプ」メニュー再実行の両方を試行。いずれも`EnumWindows`の可視ウィンドウ数は増えず（新規ウィンドウなし）、同一ウィンドウハンドルのまま`GetForegroundWindow()`が使い方ウィンドウを指すことを確認（前面化）。

### (5) `ecad2-spec-statusbar.md`由来のコンテンツがFlowDocumentとして表示されること — OK

見出し（H1「ecad2 仕様書：ステータスバー・モード可視化」、H2「1. ステータスバー全項目」等）・段落・箇条書き（「o. 要点：」）・番号付きリスト（「1.」「2.」の入れ子）・コードブロック（等幅フォント、背景色が異なる`public string SelectedCellDisplay => ...`）・水平線、いずれも正しく書式が反映されていることを目視確認（`ecad2-t077z1-dark-top.png`等）。

**隠密申し送り事項の確認**：`ecad2-spec-statusbar.md`の表構文（7行、項目/バインド先/書式/表示条件の4列×5行）は、想定どおりプレーンテキストの1段落として表示された（`| 項目 | バインド先 | ... |`という生のMarkdown記法がそのまま連続）。パイプ記号が連続し読みやすくはないが、文字化け・重なり・切れは一切なく、内容自体は視認・判読可能（`ecad2-t077z1-table-area.png`）。実害なし、PoC範囲内の既知制約として妥当。

### (6) ライト/ダークモード両方で配色が正しく追従すること（画素採取） — OK

H1見出し領域（大きく太いフォント、十分な面積の純色ピクセルが得られる箇所）で色出現頻度を集計：

| モード | 背景実測 | 背景理論値 | 文字実測 | 文字理論値 |
|---|---|---|---|---|
| ライト | `#FFFFFF`（5093px） | `#FFFFFF`（`DialogBackgroundBrush`） | `#000000`（907px） | `#000000`（`DialogForegroundBrush`） |
| ダーク | `#2D2D30`（5093px） | `#FF2D2D30` | `#F0F0F0`（914px） | `#FFF0F0F0` |

両モードとも背景色・文字色ともに理論値と完全一致。メインウィンドウの表示メニューから「ダークモード(作図色)」を切替えた際、使い方ウィンドウを開いたまま即座に配色が追従することも確認した（`SetResourceReference`によるDynamicResource機構が正しく機能）。

補足：小さいフォント（H2「1. ステータスバー全項目」等）の領域では、アンチエイリアシングの影響で純色ピクセルがほとんど得られず（背景色とアンチエイリアシング中間色のみ検出）、判定を誤りかけた。大きい見出し領域で再測定し解消——`ecad2-ui-automation`スキル既存の教訓（小さい文字・細い線は色判定に不向き）が今回も再現した形。

DoD(1)〜(6)全項目OK。範囲外の不具合検出なし。
