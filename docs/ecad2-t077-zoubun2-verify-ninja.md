「ヘルプ」メニュー「使い方(G)」項目クリックで開いたウィンドウが、左に11領域全て（シート/ドキュメント管理・メニュー・ツールバー全体構成・配置操作・結線操作・Undo/Redo・キャンバス表示・部品選択・自作パーツ管理・機器表・BOM・ステータスバー・モード可視化・設計チェック(DRC)・出力パネル・PDF出力・テストモード）を列挙したListBox、右にFlowDocumentコンテンツという構成で表示されることを確認（`ecad2-t077z2-layout.png`）。

### (2) 目次項目クリックで右側のコンテンツが切り替わること（全11領域） — OK

全11項目を順にSelectItemPatternで選択し、都度スクリーンショットを撮影して確認。いずれも目次側の選択ハイライトと右コンテンツの見出し（「ecad2 仕様書：{領域名}」）が正しく対応し、表示崩れは一切見られなかった（`ecad2-t077z2-topic0.png`〜`topic10.png`）。

### (3) ウィンドウ起動時、初期選択領域「シート/ドキュメント管理」が選択済みであること — OK

開いた直後、目次の先頭項目「シート/ドキュメント管理」が選択ハイライト済み、右コンテンツも「ecad2 仕様書：シート/ドキュメント管理」を表示していることを確認。

### (4) GridSplitterでパネル幅の調整ができること — OK

左目次パネルと右コンテンツパネルの境界にあるThumb要素（`Bounds`確認済み）を`Invoke-Ecad2Drag`でドラッグ。**初回試行はフォアグラウンド化なしで変化が見られなかったが、`Ecad2Native.SetForegroundWindow`で使い方ウィンドウを明示的にアクティブ化してから再試行したところ、目次パネル幅が約230px→約400pxへ正しく拡大され、右コンテンツの折り返しも連動して変化することを確認した**（`ecad2-t077z2-splitter-dragged2.png`）。GridSplitterのドラッグ操作にはウィンドウのアクティブ化が必要という点は、AvalonDockペインのドラッグとは異なる新しい知見として次項に記録する。

### (5) ライト/ダークモード両方で目次パネル・GridSplitter含め正しく配色されること — OK

画素採取（未選択項目・GridSplitterと重ならない座標）で確認：

| モード | 目次パネル背景 実測/理論値 | GridSplitter 実測/理論値 |
|---|---|---|
| ライト | `#FFFFFF` / `#FFFFFF`（暗黙的ListBoxスタイル） | `#D2D2D2` / `#D2D2D2`（`PanelGridLineBrush`） |
| ダーク | `#2D2D30` / `#FF2D2D30` | `#565656` / `#FF565656` |

両モードとも完全一致。メインウィンドウの「ダークモード(作図色)」切替に使い方ウィンドウが即座に追従することも確認した。

初回、目次パネルの画素採取で選択項目のハイライト帯（青系`#0E639C`）に座標が重なり誤った値を検出しかけたが、選択ハイライトと重ならない余白へ座標を移して再測定し解消した。

### (6) 表構文を含む領域（`ecad2-spec-wiring.md`等）がプレーンテキストのまま表示され実害がないこと — OK

家老指定の「結線操作」（`ecad2-spec-wiring.md`、Topics配列index=3）を含む複数領域（メニュー・ツールバー全体構成／配置操作／結線操作／機器表・BOM／ステータスバー・モード可視化／設計チェック(DRC)・出力パネル）で表構文を確認したが、いずれも増分1で確認済みの既知制約どおり、生のMarkdown表記法（`| 列1 | 列2 | ... |`）がプレーンテキストの1段落として表示されるのみで、文字化け・重なり・切れといった実害は一切なかった（DoD(2)の全11領域スクリーンショットで確認済み）。

## 範囲外の気づき（実機確認手法上の知見）

GridSplitterのドラッグ操作は、AvalonDockペインのドラッグ（`Invoke-Ecad2Drag`だけで機能、フォーカス不要）とは異なり、**対象ウィンドウのフォアグラウンド化が必要**だった。標準WPFの`Thumb`ベースドラッグ実装がマウスキャプチャに何らかの形でアクティブウィンドウ状態を要求している可能性がある。次回、標準WPFコントロール（AvalonDock以外）のドラッグ操作を検証する際の参考として`ecad2-ui-automation`スキルへの追記を提案する。
