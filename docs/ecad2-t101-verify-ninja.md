# T-101 配置ツール選択中ツールの恒久ハイライト表示 — 実機検証（忍者）

**対象コミット**: `ba0ebc7`（本実装）+ `52d71cd`（F11ハイライト維持修正）
**検証日**: 2026-07-21
**手法**: `ecad2-ui-automation`スキル。色検証は目視でなく座標指定の画素採取（支配色集計方式）で理論値と突合。

## 検証準備の補足

実機環境（マイドキュメント\Ecad2\図形）に自作パーツファイルが1件も存在しなかったため、
DoD(4)検証用に既存a接点定義をコピー・改変したテスト用自作パーツ
（`自作\T101検証用自作部品.gcadpart`）を配置した。検証専用ファイルにつき、殿の他作業に
影響なし（削除要否は殿の判断を仰ぐ。rmコマンド禁止のため忍者からは削除しない）。

## DoD別結果

### (1)(2) ツール切替の都度、対応ボタンが常時ハイライト／非選択ボタンは通常表示のまま — OK

ツールバーボタン直接クリック（`BuiltinPlaceButton_Click`経由、F5等）でa接点配置ツールへ
切替後、実測値は理論値とほぼ完全一致した。

| | 背景 実測 | 背景 理論値(alpha合成) | 枠線 実測 | 枠線 理論値 |
|---|---|---|---|---|
| ライト | `#A8CCE9` | `#A8CCE8`(alpha 0x4D の #0078D7 と #F0F0F0 の合成) | `#0078D7` | `#0078D7`(不透明) |

非選択ボタン（選択ツール・b接点配置等）は`#F0F0F0`（通常色）のまま変化なし。

### (3) 接続点記入・配線分断記入(F10系)はハイライト対象外 — OK

接続点記入は既存配線が無い状態でDisabledのため検証不可（Disabledボタンへの誤invokeは
「認識できないエラーです」という原因不明の例外を生むことを確認、6節参照）。配線分断記入
（Enabled）で検証：押下してもTool.Modeは変わらず（a接点配置のまま）、ボタン自体も
ハイライトされない（`#F0F0F0`のまま）。仕様通り。

### (4) 自作パーツ(組込でないもの)選択中もF11ボタンのハイライトが継続する — OK

テスト用自作パーツを部品選択リストから選択したところ、F11ボタンの背景・枠線は
`#A8CCE9`/`#0078D7`（(1)(2)と同一値）を維持した。`52d71cd`の修正（`ResolvePlacementToolTag()`
が`entry is null`時に`"PartSelection"`を返す）が実機で正しく機能している。

### (5) 組込部品選択時はF5〜F8等の対応ボタンへハイライトが正しく移る — 経路により結果が分かれる、判断要

**ツールバーボタン直接クリック経路（F5等）**: OK。(1)(2)で確認済み、`ActivateBuiltinTool`が
`Tool`を即座に更新するため正しくハイライトが移る。

**F11→部品選択リストから組込部品を選ぶ経路**: ハイライトが移らない（F11のハイライトが
残ったまま、対応ボタン(今回はF7=コイル配置)は非選択のまま）。コード確認の結果、これは
バグというより既存仕様との相互作用と判明した：`TryPlaceElement`
（`MainWindow.xaml.cs:3266`）→`PlacementOkButton_Click`→`PlaceElementAtSelectedCell`
（`MainWindowViewModel.cs:2786`）の一連のフローは、T-021裁定「配置後もTool/SelectedCellを
リセットしない（ツール保持で連続配置）」（`MainWindow.xaml.cs:3588`コメント）により、
そもそも`ViewModel.Tool`自体を一切更新しない設計になっている。ゆえに`ActiveToolTag`も
更新されず、ハイライトはF11のまま据え置かれる。

これがDoD(5)の意図（隠密所見2の文脈「組込部品選択時にF5〜F8等の対応ボタンへハイライトが
移る、という対称的な挙動」という記述）と一致するかどうかは忍者からは判断できない。
「F5〜F8ボタンを直接押すケース」を指しているなら達成済み、「F11部品選択リスト経由で
組込部品を選ぶケースも含む」という意図であれば未達成、という**二通りの解釈が成立しうる**。
不明な点は不明のまま報告する。実測データのみ提示し、判断は家老・隠密・殿へ委ねる。

非選択ボタン背景はダークモードのツールバー背景色`#2D2D30`と一致（ハイライトなし）。

## 隠密所見1（選択中ボタン押下時のPressedフィードバック）— 忍者からは確証困難、殿確認を推奨

瞬間的な視覚状態（マウスダウン中のみ発生するPressedフィードバック）はUIA自動化・
PrintWindow撮影のタイミング制御が粗く、既存教訓（T-089押下フィードバック検証、本スキル
6節記載）により確証が得られないことが分かっている。実装（XAMLのDataTrigger優先順位）は
隠密の静的レビューで理論確認済みだが、「選択中ボタンを押した際に押した感が全く無い」ことが
実機で許容範囲かどうかの主観的判断は、殿ご自身の実機操作でのご確認に委ねたい。
