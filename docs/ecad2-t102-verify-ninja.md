# T-102 実機確認（忍者、2026-07-24）

## 対象
`docs/ecad2-t102-or-wiring-join-target-design-onmitsu.md`（案A・1-a・2-a・解釈(i)採用）。侍実装完了・

## 操作手法
UI Automation経由のキー送信（`Send-Ecad2Keys`、F5/Shift+F5/Enter/Esc/Up/Down/矢印キー）。キャンバス内
セル選択のみ`Invoke-Ecad2CanvasClick`（座標クリック）を使用。

### 座標変換式（今回実測、既知の罠に追加）
`SelectedCellDisplay`が`$"行{pos.Row + 1}/列{pos.Column}"`という実装（行のみ+1オフセット、列は
オフセットなし）であることに気づかず、当初テストコードのGridPos座標とUI操作座標を混同し「グリッド
範囲外」の誤診断を招きかけた。正しくはUI表示の「行N」は内部Row=N-1に対応する。列方向は実測に
軽微な累積誤差があり、遠い列では都度実測確認が必要だった（列0基準からの単純な等間隔外挿では
1列程度のズレが生じた）。今回のキャンバス原点実測：列0中心=ウィンドウ相対X350、行0(内部)中心=
ウィンドウ相対Y259、列・行とも間隔30px（ただし列方向は遠方で誤差累積、矢印キーでの微調整を
併用するのが確実）。

## 観点1：単純ケース（曖昧性なし）：OK
A（通常配置）→B（OR配置、候補1件）の手順で、B確定後に自動で合流先確認モードへ遷移し、ステータス
バーに案内文言（後述）が表示された。候補は1件のみのため、Up/Down操作なしでEnter一発で確定でき、
実質無操作で完結することを確認した（画像：`t102_simple_case_done.png`）。

## 観点2：T-044実例相当：OK
`docs/ecad2-t102-or-wiring-join-target-design-onmitsu.md`のテストシナリオ
（`OrWiringTests.PlaceOr_T044Scenario_*`）と同じ構図をUI操作で再現：
A（行0列5）／B（行3列5、OR）／UNNAMED（行0列13）／COIL（行0列19）／C（行6列15、OR）。

- Cを配置しOR確定すると自動で合流先確認モードへ遷移、候補[0]（B側、近い方）のプレビュー（青い
  点線）はB行までの短い接続だった（画像：`t102_candidate0_preview.png`）。
- Downキーで候補[1]へ切替えると、プレビューがUNNAMED行（線番2側、より遠い方）まで伸びる大きな
  接続に変化した（画像：`t102_candidate1_preview.png`）——候補間で明確に異なる見た目になることを
  視覚的に確認。
- Enterで候補[1]を確定した結果、CはB(線番1)ではなくA/UNNAMED間（線番1↔線番2）に正しく合流した
  （画像：`t102_confirmed_candidate1.png`）。意図した上位ネット（線番2）への到達を確認。
- **DRC実行で裏付け**：設計チェック実行の結果、全機器（A/B/C/COIL/UNNAMED）とも「駆動元不明」
  警告のみ（コイル未配置によるテストデータ構造上当然の警告）で、断線・誤配線系のエラーは無し
  （画像：`t102_drc_result.png`）。これはCが正しくネットに接続されていることの裏付けとなる。

## 観点3：Escで要素配置ごと取消：OK
新規シートでE（通常配置）→F（OR配置、合流先確認モードへ遷移）まで進めた時点で、Fは図面上に描画
され機器表にも登録されていた（画像：`t102_before_esc.png`、機器表に「A B C COIL E F UNNAMED」の
7件）。この状態でEscを押すと、**F要素が図面から消え、機器表からも削除**（「A B C COIL E
UNNAMED」の6件に復元）された（画像：`t102_after_esc.png`）。要素配置・機器表登録の両方が正しく
ロールバックされることを確認。

## 観点4：ステータスバーの操作案内表示：OK
OR配置確定後、合流先確認モードへ遷移すると同時にステータスバーへ
「上下キーで合流先候補を切替、Enterで確定、Escで配置ごと取消」が表示された。T-041の縦コネクタ
記入時の案内文言（「上下キーで範囲、左右キー...で確定、Escで取消」）と同型の設計思想で一貫性が
ある。
