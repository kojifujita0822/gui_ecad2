## DoD別結果

### (1) 同一デバイス名の複数要素間でコメントが同期反映されること — OK

行1のM1へプロパティパネルCommentBoxから「共有コメントテスト」を入力した瞬間、行3の別M1要素（同一デバイス名）のキャンバス表示にも即座に「共有コメントテスト」が反映された（スクショで確認、`ecad2-t107z2-comment-input.png`）。続けて行3のM1を選択すると、CommentBox欄にも同一値が正しく表示された（`ecad2-t107z2-select-m1-row3.png`）。デバイス単位共有が実機で正しく機能している。

### (2) 機器表（クロスリファレンス表）のコメント列表示に回帰がないこと — OK

画面右上の機器表パネル（`DeviceTableGrid`）自体はコメント列を持たない仕様のまま（機器名/種別/型式のみ、T-107本実装時と同一構成、回帰なし）。PDF出力のクロスリファレンス専用ページ（コイル単位、2ページ目）で確認：Y1にコメント「Y1コメント」を設定後、Y1行の「コメント」列に正しく「Y1コメント」が表示された（`ecad2-t107z2-pdfpreview2b.png`）。設定前は空欄（`ecad2-t107z2-pdfpreview2.png`）で、設定後の差分も正しく反映されており回帰なし。

### (3) PDF出力・PDFプレビューに共有後のコメント値が反映されること — OK

PDFプレビュー1ページ目（ラダー図本体）で、画面表示と完全同一の内容を確認：M1（行1・行3）両方に緑色で「共有コメントテスト」、M2はコメント非表示（`ecad2-t107z2-pdfpreview1.png`）。3ページ目の部品表はコメント列を持たない既存仕様のまま（回帰なし、`ecad2-t107z2-pdfpreview3.png`）。

### (4) 永続化（保存・読込）でDevice.Commentが正しく復元されること — OK

「名前を付けて保存」で`t107z2test.gcad`として保存 → 「新規作成」でドキュメントをクリア（確認ダイアログなし＝保存済みの証跡） → 「開く」で同ファイルを再読込した結果、M1×2の共有コメント「共有コメントテスト」・Y1の「Y1コメント」・M2のコメント無し状態が全て正しく復元された（`ecad2-t107z2-reloaded.png`）。

### (5) ライト/ダーク両テーマで緑色文字（#008000）の視認性に回帰がないこと — OK

画素採取（対象領域の色出現頻度集計方式、単発ピクセル誤読対策）で確認。M1（行1）のコメント文字領域（60x13px）を走査した結果：

- ダークモード：理論値`008000`のピクセルが11px検出（背景`202226`が530pxで最多、次点はアンチエイリアシングの中間色）
- ライトモード：理論値`008000`のピクセルが11px検出（背景`FFFFFF`が525pxで最多）

両テーマで同数（11px）の理論値ピクセルが一致して検出され、視認性回帰なし。

### (6) T-107本実装分（機器名上・コメント下表示、選択切替時の通知）に回帰がないこと — OK

- 表示位置：全スクショにおいてデバイス名（M1/Y1/M2）が記号の上、コメントが記号の下に表示される既存レイアウトを維持（回帰なし）。
- 選択切替時の通知：行1のM1（コメントあり）を選択後、M2（コメントなし）へ選択切替した際、CommentBox欄が正しく空文字列に更新されることを確認（`ecad2-t107z2-select-m2.png`）。隠密指摘済みの旧`OnPropertyChanged`通知漏れ（4箇所）修正がDevice経由の参照に変わった後も引き続き実効性を保っている。
