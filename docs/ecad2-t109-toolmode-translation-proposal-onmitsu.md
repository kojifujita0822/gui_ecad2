# T-109 ToolMode日本語訳語案（隠密）

日付: 2026-07-21

## 前提の裏取り

家老の予備調査（既存ラベル5値）を`MainWindow.xaml`で再確認し、完全一致を確認した。加えて、`PlaceConnector`に対応する既存ラベルを発見した——`ToolMode.PlaceConnector`は`VerticalConnector`（縦コネクタ、制御回路シート限定）に対応し、既存ボタンラベルは**「縦分岐線記入」**（Shift+F9、`MainWindow.xaml`1231-1232行）。家老予備調査では「対応する明確な短いラベルは見当たらず」とされていたが、実際には存在する（ただし「縦分岐線」という表現は、シート種別（制御回路）に依存した文脈的な名称であり、機能名としての「縦コネクタ」とは表現が異なる点に注意）。

## 殿確認が必要な論点（UI/UX分岐、隠密からは断定しない）

### 論点1：末尾表現の統一方針

8値のうち6値は「〜記入」（縦コネクタ/グループ枠/自由線/接続点/配線分断＋今回のPlaceConnector案2採用時）で揃えられるが、`Select`（選択）・`PlaceElement`（要素配置）は動詞的表現が異なり、「〜記入」には馴染まない。`PlaceImage`も既存ラベルは「挿入」で他と不統一。

- 案A（既存ラベル最大限維持）：`PlaceConnector`=縦分岐線記入、`PlaceImage`=画像挿入のまま。既存のボタン/メニュー表記との一致を最優先。
- 案B（統一感重視）：`PlaceConnector`=縦コネクタ記入、`PlaceImage`=画像配置。ステータスバー表示としての一貫性を優先するが、既存ボタンラベルとの表現差異が生じる（同じ機能が「ボタンでは縦分岐線記入」「ステータスバーでは縦コネクタ記入」と2つの呼び方を持つことになる）。

### 論点2：`PlaceConnector`の名称選択自体

「縦分岐線記入」は制御回路シートというシート種別に依存した名称（`docs/spec/ecad2-spec-wiring.md`0節：ボタンはシート種別で切り替わる表示、モデルクラス名は`VerticalConnector`＝縦コネクタ）。ステータスバーはシート種別を問わず同じ`PlaceConnector`という一つのモードとして表示されるため、シート依存の「縦分岐線」よりも、モデル名に近い「縦コネクタ」の方が正確という考え方もありうる。

## `docs/usage/ecad2-usage-statusbar.md`との整合確認（依頼(3)）

増分6で追記した同ファイルの「ツール表示の見方」節は、意味を説明する文章（「選択モード」「要素配置モード」等、「〜モード」で統一）であり、本書の訳語案（ステータスバー実表示用の短い語）とは性質が異なる。殿裁定で訳語が確定した場合、`ecad2-usage-statusbar.md`側の説明文（「英語表記のまま表示されます」という前提の記述）も、日本語化後の表示に合わせて追って書き換えが必要になる（T-109実装後の追加采配事項として申し送る）。
