# T-109 実機検証（忍者）

**対象コミット**: `2c4d1dc`（ステータスバー「ツール:」表示の日本語化）
**検証日**: 2026-07-21
**手法**: `ecad2-ui-automation`スキル。制御回路シート・主回路シートを各1枚用意し、8種類の`ToolMode`それぞれに対応する操作を実施。
**参照**: 隠密静的レビュー`docs/ecad2-t109-review-onmitsu.md`（指摘なし）

## 結論：DoD満たすが、重要な観察点1件あり（実装の不具合ではない）

`ToolModeToTextConverter`自体は8値全てを正しく日本語化しており、コードレビュー・単体テストどおり実装は妥当。ただし**実機操作で確認したところ、8値のうち`PlaceDot`（接続点記入）・`PlaceWireBreak`（配線分断記入）の2値は、現在のコード実装上ステータスバーに表示される機会が実質的に存在しない**ことが判明した（後述、範囲外の気づきではなくDoD(2)に直結する重要な事実のため本文で報告する）。

## DoD別結果

### (1) ステータスバーの「ツール」表示が日本語で表示されること — OK

シート追加直後・各種ツール選択操作を通じて、常に「ツール: {日本語}」形式で表示されることを確認。英語表記（`Select`等）は一切見られなかった。

### (2) 実際にツール切替操作を一通り行い、対応する日本語表示へ正しく切り替わること — 6/8値を実機確認、残り2値は到達不能と判明

制御回路シート・主回路シートを用意し、以下6種類は実機で切替・表示を確認できた：

| 操作 | ToolMode | 表示確認 |
|---|---|---|
| 選択ツール(Esc) | Select | 「ツール: 選択」 |
| a接点配置(F5) | PlaceElement | 「ツール: 要素配置」 |
| グループ枠記入ボタン | PlaceFrame | 「ツール: グループ枠記入」 |
| 図面メニュー「画像挿入」 | PlaceImage | 「ツール: 画像配置」 |
| 縦分岐線記入(Shift+F9、制御回路シート、セル選択後) | PlaceConnector | 「ツール: 縦コネクタ記入」 |
| 自由線(横線)記入(F9、主回路シート、セル選択後) | PlaceLine | 「ツール: 自由線記入」 |

**残り2種類（接続点記入=`PlaceDot`・配線分断記入=`PlaceWireBreak`）は、ボタン押下後もステータスバーが「ツール: 選択」のまま変化しなかった。** 不審に思いコードを確認したところ、`TryPlaceWireBreak()`（`MainWindow.xaml.cs:3144`）は`_viewModel.PlaceWireBreakAtSelectedCell()`を直接呼んでその場で配置を完了させるのみで、`Tool.Mode`への代入を一切含まない実装だった。念のためソリューション全体を`grep`したところ、`Tool = new ToolState(ToolMode.PlaceDot, ...)` `Tool = new ToolState(ToolMode.PlaceWireBreak, ...)`という代入は**一箇所も存在しない**ことを確認した（実際に`Tool.Mode`へ代入されるのは`PlaceImage`・`PlaceFrame`・`PlaceConnector`・`PlaceLine`・`PlaceElement`の5種＋初期値`Select`の計6種のみ）。

これはT-109のバグではなく、T-041以来の既存設計（接続点・配線分断は「記入ボタンを押した瞬間に処理が完了する」仕様、`ecad2-ui-automation`スキルの既存トラブルシュートにも同旨の記載あり）に由来する。`ToolModeToTextConverter`が8値全てに対応しているのは、将来的な拡張や設計上の網羅性のためであり、現状のUIフローでは`PlaceDot`・`PlaceWireBreak`の日本語表示（「接続点記入」「配線分断記入」）を実機で目視する機会は存在しない。

### (3) 「使い方」ウィンドウのstatusbar領域の説明と実際の表示が一致していること — OK

`docs/usage/ecad2-usage-statusbar.md`（使い方ウィンドウ「ステータスバー・モード表示」領域）の「ツール表示の見方」表に記載された8種類の訳語（選択/要素配置/縦コネクタ記入/グループ枠記入/自由線記入/接続点記入/配線分断記入/画像配置）は、実装（`ToolModeToTextConverter`）の8値と完全に一致することを確認した（`ecad2-t109-usage-toolvalues.png`）。

なお、上記(2)の観察点を踏まえると、この表の「接続点記入」「配線分断記入」の2行は理論上の説明として正確ではあるものの、実機でユーザーがこの文言を実際に目にする場面は無い（ボタン押下後は常に「選択」のまま）。ドキュメントの記載自体に誤りはないが、参考情報として申し送る。
