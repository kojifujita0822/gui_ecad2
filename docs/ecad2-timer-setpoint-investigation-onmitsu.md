# タイマー接点テストモード未反映 実態調査（隠密）

調査日: 2026-07-15
調査者: 隠密
委任元: 家老（殿直接指摘「タイマー接点のテストモード時の挙動が反映されていない」）

## 結論

**「Core完備・App未結線」パターンの新しい一形態と判断する。** ただし従来（T-060/T-061/T-086等）と異なり、機能全体の未結線ではなく、**1パラメータ（タイマ設定時間`Setpoint`）だけがApp層に書き込み経路を持たない**という粒度の細かい欠落。この欠落により、限時タイマ接点はテストモード中ずっと固定表示のまま変化しない。

## 根拠

### (a) Core層のタイマーシミュレーションロジックは完備

- `src/Ecad2.Core/Simulation/Evaluator.cs:145-185`（`IsConducting`）：瞬時接点(152-158行)・限時接点(161-169行)とも実装済み。限時接点は`_net.TimerSetpoints`（設定時間）と`s.TimerElapsed`（経過時間）を突き合わせ`timedOut = sp > 0 && elapsed >= sp`で判定。
- `src/Ecad2.Core/Simulation/TestSession.cs:46-63`（`Tick(dt)`）：励磁中のタイマコイルの経過時間加算、消磁時リセット。実時間計時本体。
- `src/Ecad2.Core/Simulation/NetlistBuilder.cs:295-338`：`Params["Setpoint"]`を読み`Netlist.TimerSetpoints`を構築。
- 注：Core層にオフディレイ（TOF）タイマーの区別は無い（限時a/b・瞬時a/bの4種のみ、`ElementKind`にオン/オフディレイ区別なし）。

### (b) App層：テストモード機能自体は結線済みだが、Setpoint入力経路が皆無

- テストモード切替・実時間Tick（`DispatcherTimer`100ms、`MainWindow.xaml.cs:330-358`）・通電色分け描画（`DiagramRenderer.cs:239-261,944-982`）は結線済み。従来型の「機能まるごと未結線」パターンには該当しない。
- **しかし`grep "Setpoint" src/Ecad2.App` → 0件。** プロパティパネル（`MainWindow.xaml:595-614`）には「デバイス名」「ノッチ位置」（T-086、セレクトSW限定）欄のみで、タイマ設定時間欄は存在しない。`SelectedElementNotchPosition`（`MainWindowViewModel.cs:1854-1878`）に相当する`SelectedElementSetpoint`は無い。タイマコイル専用の`PartDefinition`も無く、汎用「コイル」を同名デバイスとして流用する構造。

### 決定的な帰結（論理的帰結、推測ではない）
`Evaluator.cs:165`の`TimerSetpoints.TryGetValue(...)`は書き込み経路が無いため常に失敗し`sp`はdouble既定値0.0。`timedOut = sp > 0 && ...`（167行）は常にfalse。結果、**限時接点NOは常に非導通（グレー固定）、限時接点NCは常に導通（赤固定）——コイルの励磁状態や経過時間に一切関係なく、テストモード中ずっと固定表示のまま変化しない**。これが殿ご指摘の症状と一致する根本原因と判断する。

### (c) T-061のDoDにタイマ接点の限時動作は含まれていなかった

`docs/todo-archive.md:1230-1285`（T-061完全Done節）確認。タイマ関連の記述は「実時間タイマパネル・ステータス連携」（Tick機構と一時停止ボタン）のみで、Setpoint入力UIやタイマ接点の導通判定の検証観点には一切言及がない。T-061レビュー時に発見されたB-2バグ（`docs/archive/ecad2-t061-review-onmitsu.md`、限時タイマ接点が時期尚早に赤くなる）は色分けロジック自体のバグで修正済みだが、これはCore層ユニットテストで`Params[Setpoint]`を直接注入した検証であり、実機UIでの再現・Setpoint入力経路の有無とは無関係。T-061のDoD5段階（モード切替/通電色分け/入力操作/右クリック強制ON-OFF/実時間タイマパネル）にSetpoint入力は含まれていない。

### (d) 既存の実機確認記録にも記載なし

T-071実機確認記録（限時/瞬時接点の部品配置・ラベル表示確認のみ、`docs-notes/ecad2-t071-realmachine-verification-ninja.md`）はタイマの静的表示確認に留まり、テストモード中の導通挙動（経過時間に応じた色変化）は検証対象外だった。他の忍者実機確認記録（T-061 round1/round2）にもタイマ関連の記述は0件。

## 類似先例との関係

T-086（セレクトSWの`Position`書き込み経路皆無、`docs-notes/ecad2-t086-position-ui-absence-samurai.md`）と根本原因が完全に同型（1パラメータの書き込み経路自体が存在しない）。パターン台帳（PR-01〜PR-15）には正式記帳されていない（T-086も未記帳）ため、この調査結果を機に新規パターン候補として提示する価値がある（要家老判断）。

## 殿情報との整合

殿より「GuiEcadにはタイマー時間設定（MAX10秒）で設定でき作動できたはず」との情報を頂いた。ecad2はGuiEcad実ソースを全層移植（T-007）しているため、Setpoint入力UI自体はGuiEcad原本に存在していた可能性が高い（未検証、推測）。P-086（セレクトSWノッチ番号、GuiEcad原本の実装形式調査から着手）と同型の段取りが妥当と考えられる。
