## 1. PDF出力（T-060、完全Done）

### 実行方法

メニュー「ファイル」→「PDF出力(_P)」（Ctrl+P）、またはツールバーボタンから
`PdfExportMenuItem_Click`（`MainWindow.xaml.cs`1092-1104行）が呼ばれ、回路番号採番・
CrossReference構築後、**モーダルの`PdfPreviewDialog`を表示**する2段階UI（殿裁定2026-07-12）。

### プレビューダイアログの操作

`PdfPreviewDialog.xaml`/`.xaml.cs`：

| 操作 | ボタン/挙動 |
|---|---|
| ページ送り | 「◀」`OnPrev`／「▶」`OnNext` |
| ズーム | 「－」`OnZoomOut`／「＋」`OnZoomIn`（0.25刻み、範囲25%〜400%） |
| 幅に合わせる | 「幅に合わせる」`OnFitWidth`（ズームを100%へリセット） |
| PDF出力実行 | 「PDF出力」ボタン`ExportButton_Click` |
| 閉じる（何も出力しない） | 「閉じる」ボタン、`IsDefault="True" IsCancel="True"`——**Enterキーの既定動作は安全な「閉じる」**（出力実行ではない） |

### 出力範囲・枠の扱い

- **出力範囲は常に全シート**（現在シートのみの選択肢は無い、殿裁定2026-07-12）。
- 枠は`DocumentSettings.EnableBorder`の値をそのまま使用（`PdfExporter.cs`23行）、**UIでの切替は
  無い**——プレビューにも同じ`enableBorder`が反映される。

### 保存先の指定

`ExportButton_Click`内で`SaveFileDialog`（フィルタ`*.pdf`、既定ファイル名はドキュメントタイトルまたは
"diagram"）を表示、確定後に`PdfExporter.Export(document, library, path)`を呼ぶ。出力内容は
全シートページ＋クロスリファレンスページ＋BOM（部品表）ページ（`docs/spec/ecad2-spec-device-table.md`
0節参照）。例外時は日本語固定文面のメッセージボックスで通知。

## 2. テストモード（T-061、完全Done）

### モード管理

`AppMode` enum（`ToolState.cs`34行）：`Drawing`/`Test`の2値（design-brief準拠の上位enum一元管理、
GuiEcadの単純boolフラグから設計改善）。`IsTestMode`は`Mode==AppMode.Test`のget/setラッパー。

**起動方法**：メニュー「ツール」→「テストモード」（Ctrl+T、`IsChecked`双方向バインド）、または
ツールバー「基本機能」タブのToggleButton（`docs/spec/ecad2-spec-menu-toolbar.md`2節参照）。

### モード遷移時の挙動

`Mode`セッタ（`MainWindowViewModel.cs`89-120行）：
- ON/OFFいずれの遷移でも`_testSessions`（シートごとのテストセッション辞書）を**全クリア**
  （GuiEcad踏襲、殿裁定＝OFF時に全シート分クリア）。
- Test化時：進行中のドラフト（縦コネクタ/自由線/画像挿入）を明示クリア、`Tool`を
  `ToolState.SelectDefault`へ固定、現在シートがあれば`Evaluate()`実行。
- `CanEditDiagram => HasProject && Mode==AppMode.Drawing`——**テストモード中は編集操作全般が
  無効化される**（`docs/spec/ecad2-spec-undo-redo.md`4節参照）。`Tool`セッタ自体もテストモード中は
  `ToolMode.Select`以外への変更を拒否する二重ガードを持つ。

### 入力操作（機器種別ごとに反応が異なる）

| 機器種別 | 操作方式 |
|---|---|
| 押しボタン（`DeviceClass.PushButton`） | **モーメンタリ**（マウスキャプチャで押している間だけON、`PreviewMouseLeftButtonUp`/`LostMouseCapture`でOFFへ戻す） |
| セレクトスイッチ | クリックで`CycleSelectSwitch`によりノッチ順送り |
| a接点・b接点（真の接点） | クリックで`ToggleInput`によりトグル |
| コイル・電磁接触器主接点・ランプ・端子台・タイマ・カウンタ等 | 無反応 |

キーボードEnterもマウス左クリック相当として同じ入力処理を共用する。

### 右クリックメニュー（真の接点限定）

真のa接点/b接点のみ対象。「接点: {機器名} [手動ON中/シミュレーション依存]」（無効表示）・
「手動でON(強制閉路)」・「手動を解除(シミュレーション依存に戻す)」の3項目。

### 通電表示の配色

`DrawingTheme.cs`：**通電/励磁＝橙色**（`Powered`、「LDmicro式＝通電赤/非通電グレー」を踏まえつつ
殿要望によりオレンジ色に調整）、**非通電＝グレー**（`NonEnergizedGray`）。通電時は線幅も強調
（`PoweredWireWidth`）。手動強制中は半透明の青系の塗りで区別（`ManualForced`）。

### 実時間タイマ

専用の操作パネルは無い。モード遷移に連動して`DispatcherTimer`（100ms間隔）が自動的に開始/停止し、
励磁中のタイマコイルの経過時間を裏側で加算・評価してキャンバスへ反映する。ユーザーが操作できる
ボタン等は無い（当初検討の「タイマ一時停止」機能は殿裁定により削除・対象外）。

## 4. 既知の罠・修正済みバグ（T-061往復レビューで発見・修正済み）

- **セレクトSWの電気的導通判定が構造的に機能しなかった**（`PartRole`enumに`SelectSwitch`値が
  存在せずマッピングパスが構造的に欠落）→根本原因を特定し構造対処（`PartResolver`/
  `NetlistBuilder`設計変更）で修正済み。既存展開済み環境（開発機含む）への自動反映も
  マイグレーション処理で対応済み。
- **テストモード中に画像挿入メニューをクリックすると`HasAnyDraft`が恒久true化し、テスト用
  右クリックメニューが二度と出せなくなる重大バグ**→修正済み（画像挿入等の`CanEditDiagram`
  ガード漏れが原因）。
- テストモード突入直後、セレクトSWの全ノッチが一時的に非導通表示になる現象（クリックした瞬間
  から正しくなる）→殿確認済み、対処不要と判断され決着。
