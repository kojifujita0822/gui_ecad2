# ecad2 仕様書：ステータスバー・モード可視化

T-075（殿裁定、2026-07-11起票）体系の第11号、第4弾最終6件目。実装コード・殿裁定記録
（`docs/todo.md`/`docs/todo-archive.md`/`docs/proposed.md`）・忍者実機検証記録（`docs-notes/`配下）を
突き合わせ、「仕様として確定している挙動」を出典付きで明文化する。**これにてT-075全11領域の
仕様書化が完了する。**

---

## 0. 要点：現在アクティブなツールはツールバーの恒久ハイライトとステータスバーの両方で確認できる

**（2026-07-21更新、T-101反映）** T-101以前は、ツールバーボタンにハイライト表示が存在しなかった
——配置ツールボタン群は全て素の`Button`（`ToggleButton`ではない）で、`Tool.Mode`連動の
`Background`/`Style`トリガーが無く、唯一の視覚状態変化は`IsEnabled`に連動した半透明化
（opacity 0.35、「選択中か」ではなく「押下可能か」を示すのみ）だった。

**T-101（2026-07-21、コミット`ba0ebc7`）で新設**：現在有効なツール（`Tool.Mode`から算出される
`ActiveToolTag`、`MainWindowViewModel.cs`）に対応する配置ツールボタン
（`PlacementToolBarButtonStyle`適用群、F5〜F9・グループ枠記入・自作パーツ等）を、カーソル位置に
依存せず背景色+枠線で恒久的にハイライトする（新規`DynamicResource`
`PlacementToolActiveBackgroundBrush`/`PlacementToolActiveBorderBrush`）。ただし即時実行で
`Tool.Mode`を変更しない接続点記入・配線分断記入（F10系）は恒久ハイライトが原理的に成立しない
ため対象外——これらは従来どおりステータスバーの「ツール: {Mode}」テキストが唯一の確認手段の
まま。詳細は`docs/ecad2-t101-review-onmitsu.md`（隠密静的レビュー）・`docs/ecad2-t101-verify-ninja.md`
（忍者実機確認）参照。

同様に**シート種別（主回路/制御回路）を示す表示もどこにもない**——`Sheet.MainCircuit`はF9/F10系
ボタンの`IsEnabled`切替にのみ使われ、テキストとしては一切表示されない。

## 1. ステータスバー全項目

`MainWindow.xaml:555-577`、`StatusBar x:Name="StatusBarArea"`内、`Separator`区切りで最大5項目：

| 項目 | バインド先 | 書式 | 表示条件 |
|---|---|---|---|
| ツール | `Tool.Mode` | `'ツール: {0}'` | 常時 |
| 選択セル | `SelectedCellDisplay` | `'選択セル: {0}'` | 常時 |
| 操作対象端点 | `SelectedEndpointDisplay` | `'操作対象: {0}'` | `HasSelectedLinePrimitive`の時のみ表示 |
| ズーム | `CanvasScale` | `'ズーム: {0:P0}'` | 常時 |
| 案内/警告メッセージ | `StatusMessage` | プレーンText、`Foreground="DarkRed"`固定 | 常時（空文字なら実質非表示） |

XAML冒頭コメント：「design-brief 4節#6『モード可視化の多重化』。現在の配置ツールモードとズーム率を
表示する最小実装。GX Works3同等の高情報密度（機種名/局番/ステップ数等）は本実装向け機能に依存する
ため将来タスク」——これが**T-022（ステータスバーの高情報密度化）**として切り出され、現在も
**Proposed（gated、未着手）**のまま。

## 2. `StatusMessage`（一時的な案内メッセージ）

`MainWindowViewModel.cs:1322-1329`、単純な`get/set`。セット箇所は`MainWindow.xaml.cs`に集中し、
4パターンに分類できる：

1. **ツール起動時の操作案内**（例：「配置ツール: {partName}{OR} - キャンバスをクリックして配置
   位置を指定してください」）——コメントに「ゴースト（プレビュー）表示は簡易版としてステータス
   バー表示のみに留める（視覚プレビューはT-029、Rejected）」とあり、**`StatusMessage`が未実装の
   視覚プレビューの代替を兼ねている**設計意図が読み取れる。
2. **前提未達エラー**（「シートがありません。新規作成（Ctrl+N）から始めてください」「配置する
   セルを先に選択してください」「縦分岐線の記入は制御回路シートでのみ使用できます」等）
3. **競合/重複エラー**（「この位置には既に配線分断があります」等）
4. **クリア**（空文字への明示リセット）

## 3. ツール名表示ロジック

**コンバータは存在しない**。`Tool.Mode`（`ToolMode` enum：`Select`/`PlaceElement`/
`PlaceConnector`/`PlaceFrame`/`PlaceLine`/`PlaceDot`/`PlaceWireBreak`/`PlaceImage`、
`ToolState.cs`12行）は`StringFormat`のみで直接文字列化される。**「ツール: 」の接頭辞のみ
日本語化されており、値部分は英語enum名がそのまま表示される**（例：「ツール: Select」
「ツール: PlaceConnector」）——日本語変換ロジックは存在しない。**（2026-07-21追記）旧版は
`PlaceImage`（T-064、画像挿入機能、2026-07-13追加）が列挙から漏れていた（欠落）。**

## 4. 選択セル表示

```csharp
public string SelectedCellDisplay => SelectedCell is { } pos ? $"行{pos.Row + 1}/列{pos.Column}" : "未選択";
```

**「行N列M」ではなく「行N/列M」（スラッシュ区切り）**。かつ**RowのみRow+1（1始まり表示）、
Columnは非加算のまま（0始まり表示）**という非対称な仕様——コード上の事実であり、この非対称性の
意図はコメントに記載なし（**不明点**）。未選択時は`"未選択"`。

## 5. 線分端点操作対象の表示

`SelectedEndpointIsStart`（bool、既定`true`=始点）→`SelectedEndpointDisplay`
（`"始点"`/`"終点"`）。`HasSelectedLinePrimitive`（`SelectedConnector`または`SelectedFreeLine`が
非null）でステータスバー項目自体の可視性を判定。

### 実機発覚の教訓（コード内コメントに明記）

`SelectedEndpointIsStart`のsetterは`SelectedEndpointDisplay`への変更通知を**明示的に発火する必要が
ある**——`SetProperty`は自身の変更通知のみ発火し、派生表示プロパティへは自動伝播しない。これを
忘れると`ToggleSelectedEndpoint()`は内部的に正しく動作するのにステータスバー表示だけ更新されない、
という気づきにくい表示バグになる（実機で「Tabが効かない」ように見えた、という教訓がコメントに
残されている）。

## 6. モード可視化の他の手段（0節の詳細）

- ツールバーボタンのハイライト：**T-101（2026-07-21）で新設**、配置ツールボタン群のみ対応
  （接続点記入・配線分断記入は対象外、0節参照）。
- シート種別（主回路/制御回路）の表示：**なし**。左パレットのシート一覧は`DisplayMemberPath="Name"`
  でシート名のみ表示、種別はテキストとして描画されない。種別選択のUIはシート追加ダイアログの
  ラジオボタンのみ。

## 7. 裁定根拠

### 初出実装（T-009）と未着手の残課題（T-022）

ステータスバーの初出はT-009（骨格UI、全8段階、Done）。ステータスバー固有の高情報密度化は
**T-022として切り出され、現在もProposed（gated）＝未着手のまま**残存（「機種名/局番/ステップ数
など他機能への依存があり、依存元の実装状況次第」）。

### T-016「複数ツールボタン同時ハイライト・切替不能」バグ（修正済み）

2026-07-03、忍者のT-016実機検証中に発生。**原因はToolStateがrecord structのため、`SetProperty`の
構造的等価性チェックで「同じ図形の再選択」等の際に値が変わらないと判定されPropertyChangedが
発火せずUIが更新されないバグ**（`ecad2-ui-automation`スキルの既知の罠にも記載）。修正済み：
`MainWindowViewModel.cs`の`Tool`プロパティで、コメント付きで意図的に等価性チェックをバイパスし
常に通知する実装に変更（`94a4e65`）。この教訓は`TraceLog.cs`にも引用され、以後の検証でも既知の
罠として繰り返し参照されている。

### ツールバーボタンのハイライト表示自体の専用殿裁定（T-101で確定）

近縁のタスクはT-026（ツールバー7ボタン化）・T-040（GX様式の意匠確定）だが、いずれも**アイコン
意匠・配置**に関する裁定であり、「選択中ボタンをどう強調表示するか」自体を主題とした裁定は
長らく不在だった。**T-101（2026-07-21）で解消**：殿直接指示を契機に調査したところ元々の仕様
欠落（T-040導入時点から恒久ハイライト機構自体が存在しなかった）と判明し、殿裁定（視覚表現＝
背景色＋枠線、対象範囲＝配置ツールボタン群のみ）により恒久ハイライトが新設された（0節参照）。

なお選択セルの視覚ハイライト（キャンバス内の枠線）はT-027（T-017に統合、Done）であり、本仕様書の
対象（ツールバーボタンのハイライト）とは別物。

### `docs/proposed.md`の直接該当なし

近縁のP-023（部品選択リストの選択ハイライトが残らない）はT-054として起票・完全Done済みだが、
対象は部品選択リストでありツールバーボタン・ステータスバーとは別機能。
