# GuiEcad仕様書：配置操作

T-081（殿直接指示、2026-07-12起票、隠密2指名）体系。GuiEcad原本
（`C:\Users\kojif\Desktop\生産物\gui_ecad\`）の配置操作実装をExplore委譲調査で纏め、
`docs/spec/ecad2-spec-placement.md`（ecad2側、T-075起票）と比較可能な形で整理する。

対応するecad2側仕様書：`docs/spec/ecad2-spec-placement.md`

## 1. 配置操作（キー割当・マウス操作）

**F5〜F8はキー1発で「ツール切替」のみ**を行う（押した瞬間に要素は生成されない）。F5=a接点、
F6=b接点、F7=コイル、**F8=押しボタンNO**（`MainPage.KeyBindings.cs:95-102`）——**ecad2の
「F5〜F8=a接点/b接点/コイル/端子台」とは対象が異なる**（端子台は含まれない）。

実際の配置は**キャンバス上のマウスクリック**で確定。`PlaceKind`が立っていればクリック位置の
セルへ即座に`PlaceElementAt`（`MainPage.Pointer.cs:303-308`）。

**キーボード配置モード**という独立モードも存在（`MainPage.KeyboardMode.cs:11-92`）：トグルON後、
矢印キーでフォーカスセル移動、数字キー1〜9・0でツール切替（1=a接点…9=端子台、0=選択）、
Enterでフォーカスセルへ配置。この数字キー体系がecad2のF5〜F8に近い「一発配置寄り」の操作系だが、
対応キー・操作系統ともF-keyとは別物。

「その他部品」ドロップダウンから常設外の記号・自作パーツを選択してツール化も可能
（`OnOtherPartSelected`）。選択後はキャンバスクリックで確定。

**OR配置に相当する機能は存在しない**。`IsOr`/`OrEligible`等の語はGuiEcad全ソースで検索0件、
Shift+F5/F6の割当も`CommandDefs`に存在しない。

## 2. 確定・取消操作

**配置バー（ElementPlacementBar）に相当するUIは存在しない**。クリック（またはキーボード配置
モードのEnter）した瞬間に`_history.Execute(new PlaceElementCommand(...))`が実行され、**即座に
確定・Undo履歴へ登録される**（`MainPage.Pointer.cs:119-130`）。デバイス名等を確認する中間ステップ
はない。

配置後の機器名入力は別経路：マウス配置は選択状態のまま（何もしない）、Enterキーで選択中要素の
機器名インライン編集が起動。キーボード配置モードでは配置成功直後に**自動で**機器名エディタが
開く（`MainPage.Menu.cs:49-50`）——マウス配置とキーボード配置で「名前入力の自動起動有無」が異なる。

**「確定」概念自体がない**（クリック=即時コミット）ため、Escでの「配置取消」も存在しない。
Escapeはツールを選択ツールへ戻すだけ。すでに置いた要素の取消は**Ctrl+Z（Undo）**のみ。

**重要な差異**：ecad2は「要素配置はUndo対象外」が元の仕様だが、**GuiEcadは配置が
`PlaceElementCommand`としてUndo/Redo対象になっている**。

## 3. 部品選択パレット経路

ecad2のような「右パネル下段が部品選択リストに切替わる」専用UIは存在しない。GuiEcadでの自作
パーツ・追加記号アクセスは**フロートツールパレット内「その他▼」**または**トップメニュー
「図形(G)」**の2箇所（内容はほぼ共有、`guiecad-spec-part-management.md`5節参照）。

自作パーツ・自作図形1件ごとに「配置」「編集...」「削除」のサブメニューを持ち、**パーツ編集・
削除がメニューから直接可能**（ecad2はT-068未着手でエディタUI自体がない）。クリックで選ばれるのは
「配置対象として`_tool`をセット」だけで、その後キャンバスクリックで確定する2段階操作——ecad2の
「ツール選択→クリックで確定」経路に近い挙動。

## 4. バリデーション・エラーメッセージ

| 制約 | GuiEcadでの実装 |
|---|---|
| 挿入位置が既存要素の途中 | 拒否、「既存の要素の途中には挿入できません。」（`MainPage.Pointer.cs:93-97`） |
| 挿入により行末をはみ出す | 拒否、「行末をはみ出すため挿入できません。」（同103-106） |
| 未選択セル | 該当なし（セル選択という概念自体がなく、クリック位置がそのまま配置先） |
| プロジェクト/シート未作成 | 該当なし（`MainPage()`コンストラクタで必ず初期化、シート0件状態が存在しない） |
| **重複配置（占有セル）** | **拒否ではなく既存要素を右へシフトして挿入**する方式。シフト結果が行末を超える場合のみ上記2番目のエラーで拒否 |
| ドラッグ移動時の移動先重複 | `CellEmpty`が`CellWidth`考慮の範囲でチェック、移動不可時はエラーメッセージなしで位置更新をスキップ |

**ecad2との構造的な違い**：ecad2は「占有セルへの配置を拒否」方式（`Pos`完全一致のみ判定、
`CellWidth`未考慮）。GuiEcadは逆に**「挿入＝既存要素を押し出す」方式**であり、かつ移動時の重複
判定は`CellWidth`を考慮する点でecad2より厳密。「重複配置」という同じ言葉が指す実装がまったく異なる。

シート種別（`MainCircuit`）によるF5〜F8等のツール切替ゲートは確認できなかった——ecad2と同じく
「配置操作にシート種別チェックがない」という共通点（GuiEcad側は検索結果ヒットなしに基づく判断、
実機未検証のため推測を含む）。

## 5. データ経路の統一状況（ecad2との根本的な差異）

**GuiEcadは組込み品と自作パーツでデータ経路が統一されていない。** ecad2とは対照的な設計。

`ElementInstance`は`Kind`（enum、配置時に実際に意味のある値がセットされる）と`PartId`
（nullable、自作パーツのときのみ設定）の両方を持つ——「`PartId`がnullなら組込み種別(`Kind`)」
（`Element.cs:53`）。F5〜F8・数字キー・組込みその他記号は`Kind`のみ設定し`PartId=null`のまま。
自作パーツ・組込みパーツ・自作図形は`PartId`を設定し`Kind`は無視される既定値で埋める。

`PartResolver`の「`PartId`があればLibrary解決、無ければ`Kind`ベースのCatalog解決」という
フォールバック構造自体はecad2と似ているが、**`Kind`が死んだフィールドではなく実際に使われる
一次データ**である点が本質的に異なる。結論：GuiEcadは「組込み記号=Kind直結」「自作/組込みパーツ
=PartId経由」の**二系統構造**であり、ecad2の「全経路がPartId経由に統一」とは根本的に異なる
アーキテクチャ。

`BasicPartTemplates`という同名クラスが存在するが用途は別物（自作パーツエディタの「たたき台」
テンプレートのみ、F5〜F8等の実配置経路には無関係、基本図形の実フォルダシードは廃止済み）。

## 6. GuiEcadとecad2の比較（一覧）

### (1) GuiEcadのみにある機能

| 機能 | 出典 |
|---|---|
| キーボード配置モード（矢印+数字キー+Enter、自動機器名編集起動） | `MainPage.KeyboardMode.cs:11-92` |
| 挿入＋自動シフト配置（既存要素を右へ押し出す） | `MainPage.Pointer.cs:79-136` |
| 配置＝即時Undo可能（`_history.Execute`） | `MainPage.Pointer.cs:119-130` |
| 移動時の`CellWidth`考慮重複判定 | `MainPage.xaml.cs:663-672` |

### (2) ecad2のみにある機能

| 機能 | ecad2側出典 |
|---|---|
| OR配置（Shift+F5/F6、縦コネクタ自動生成） | `ecad2-spec-placement.md`3節 |
| 配置バーによるOK/Esc確定・取消（原子的、要素はOK確定時のみ生成） | 同4節 |
| F5〜F8=端子台を含む4種直接配置 | 同0節 |
| 部品選択パレットの右パネル切替式リスト＋サムネイル | 同5節 |
| 配置がUndo対象外という明示仕様 | 同9節 |
| プロジェクト未作成時のガード＋案内メッセージ | 同6節 |

### (3) 両方にあるが挙動が異なる点

| 項目 | ecad2 | GuiEcad |
|---|---|---|
| F5〜F8の意味 | 一発配置（選択済みセルへ即配置バー表示） | ツール切替のみ、配置は別途クリック必要 |
| F5〜F8対象4種 | a接点/b接点/コイル/端子台 | a接点/b接点/コイル/押しボタンNO |
| 重複配置時の挙動 | 拒否のみ | 既存要素を自動シフトして挿入 |
| 配置のUndo可否 | Undo対象外 | Undo/Redo対象 |
| データ経路 | `PartId`に統一（`Kind`は死んだフィールド） | `Kind`(組込み記号)と`PartId`(自作/組込みパーツ)の二系統 |
| 配置確定の操作 | 配置バーのOKボタン | クリック即時確定 |
| 配置取消の操作 | Escで配置バーを閉じる（原子的） | Ctrl+Z（Undo）のみ、Escはツール切替のみ |
