## 1. ステータスバー相当UIの有無・構成

存在する（`MainPage.xaml:821-869`、`Border`+`Grid`+`StackPanel`の自前構成、ecad2のような
`StatusBar`コントロールではない）：

| 要素 | 内容 | 常時表示か |
|---|---|---|
| `StatusMode` | `"作画モード"`/`"テストモード"` | 常時 |
| `StatusPos` | `"行: {row+1}  列: {col+1}"` | 常時（既定`"行: -  列: -"`） |
| `StatusSheet` | `"シート {idx} / {total}"` | 常時 |
| `StatusDirty` | `"変更済み●"` | 未変更時Collapsed |
| `StatusWarn` | DRC/短絡警告の連結文字列 | 警告有無で表示切替 |

ecad2の5項目（ツール／選択セル／操作対象端点／ズーム／案内メッセージ）とは**項目構成が根本的に
異なる**。

## 2. ツール状態の表示方法（ecad2と役割分担が逆転）

**テキストとハイライトの両方が存在**——ecad2の「テキストのみ」設計とは正反対。

- ハイライト：配置ツールパレットは`RadioButton`集合、選択中ボタンの背景がアクセント色に切替わる
  カスタム`ControlTemplate`（`MainPage.xaml:14-89`）。GuiEcadの`ToolMode`
  （`Select/PlaceElement/PlaceConnector/PlaceFrame/PlaceLine/PlaceDot/PlaceWireBreak`）はecad2の
  `ToolMode` enumと値が一致——ecad2がGuiEcadの列挙をそのまま継承したと見られる。
- テキスト：`StatusMode`は**ツール名を表示しない**。「作画モード」/「テストモード」の二値のみ
  （`MainPage.xaml.cs:418-433`）。ツール名は**キャンバス下部の`HintText`**（出力パネルフッター）に
  操作ヒント文として間接的に現れるのみで、ツール名そのものはパレットのラベルテキストでしか
  判別できない。

まとめると、GuiEcadは「ボタンハイライト」で選択中ツールを示し「ステータスバーのテキスト」は
作画/テストの区別のみ——**ecad2の「テキストのみでツール名を示す」設計とは役割分担が逆転**。

## 3. 選択セル位置の表示形式（ecad2と書式・意味論の両方が異なる）

`$"行: {cell.Row+1}  列: {cell.Column+1}"`（`MainPage.xaml.cs:485-490`）。

- 書式：コロン+半角スペース区切り（ecad2の`"行N/列M"`スラッシュ区切りとは異なる）。
- **Row・Column両方+1の対称表示**——ecad2の非対称仕様（Rowのみ+1、Columnは非加算）とは異なる。
- 未選択時は`"行: -  列: -"`（ecad2の`"未選択"`とは表現が異なる）。
- **本質的な違い**：この値は「選択セル」ではなく**マウスホバー中のグリッド座標をリアルタイム
  追跡**したもの（`OnPointerMoved`で常時`UpdateStatusPos`呼び出し）。クリックによる「選択」の
  永続表示ではなく、ポインタ移動のたびに値が変わる——ecad2の`SelectedCellDisplay`（選択解除まで
  保持）とは意味論が異なる。

## 4. 案内・エラーメッセージの表示場所（3チャネルに分離）

**ステータスバーには一切ない**——ecad2の`StatusMessage`（常時表示領域の赤字）に相当する項目は
GuiEcadのステータスバー5要素に存在しない。GuiEcadは目的別に3系統：

1. **操作ヒント**：`HintText`（出力パネルフッター、Opacity 0.55）。`UpdateHintText()`が
   `(_testMode, _tool.Mode)`＋選択状態で操作ヒント文言を切替（例：「クリック: 配置 | 右クリック:
   メニュー | Esc: 選択に戻る」）。常設のヒント帯であり一時メッセージではない。
2. **警告**（テストモード中のDRC/短絡）：`StatusWarn`（ステータスバー右寄せ）。作画モード中は
   常に非表示。
3. **エラー**：**モーダル`ContentDialog`**（`ShowErrorAsync`、`MainPage.Menu.cs:567-584`）。
   ファイル読込失敗・キーバインド競合・エクスポート対象なし等、十数箇所から呼ばれUIをブロックする。

ecad2の「非モーダル・単一チャネル・常時見える赤字」設計に対し、GuiEcadは3チャネルに分離。

## 5. モード可視化の他の手段（GuiEcad固有が多数）

- **配置プレビュー（ゴースト）**：配置ツール選択中、マウス位置に半透明シンボルを描画
  （`MainPage.Drawing.cs:144-160`）。**ecad2は同種の視覚プレビューをT-029として明示的にRejected**
  し`StatusMessage`テキストが代替を兼ねる裁定——GuiEcadとecad2で正反対の設計判断。
- ドラッグ中の作画プレビュー（縦コネクタ・自由線・枠の仮図形表示）。
- キーボード配置モードのフォーカスセルをオレンジ枠で強調。
- **ツールバー背景色**によるテスト/作画モード切替（`AppTestModeBrush`、ライト`#FFFFEBC8`／
  ダーク`#FF2D1A00`）。
- **タイトルバーのダーティフラグ**（`"<ファイル名>[ *] - GuiEcad"`）——ステータスバーの
  `StatusDirty`と二重にダーティ状態を提示。
- ステータスバー背景色自体はモードで変化しない（変わるのはツールバー側のみ）。

| 常設の操作ヒント帯（`HintText`） | `MainPage.xaml:694-695` |

### (2) ecad2のみにある機能

| 機能 | 備考 |
|---|---|
| ツール名のステータスバー直接表示（`"ツール: {Mode}"`） | GuiEcadはStatusModeが作画/テストの二値のみ |
| ズーム率のステータスバー常時表示 | GuiEcad主キャンバスにズーム%表示なし |
| 操作対象端点（始点/終点）の表示・Tab切替 | GuiEcadに該当機能なし |
| 案内メッセージのステータスバー内蔵（非モーダル） | GuiEcadは常設ヒント帯＋モーダルダイアログに分離 |

### (3) 両方にあるが挙動が異なる点

| 項目 | ecad2 | GuiEcad |
|---|---|---|
| 選択セル位置の書式 | `行N/列M`、Rowのみ+1の非対称 | `行: N  列: M`、両方+1の対称 |
| 選択セル位置の意味論 | クリック確定の選択を保持 | マウスホバーのリアルタイム座標 |
| 現在ツールの可視化 | ステータスバーテキストのみ | ボタンハイライトのみ |
| エラー/警告の表示方式 | ステータスバー内非モーダル赤字 | エラー=モーダル、警告=テストモード限定ステータスバー |
| 未保存状態の表示 | ecad2仕様書に記載なし | ステータスバー＋タイトルバーの二重表示 |
| 配置中の視覚プレビュー | T-029でRejected（テキストのみで代替） | ゴーストプレビュー実装済み |
